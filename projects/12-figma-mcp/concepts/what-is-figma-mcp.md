# Figma MCP란? — 건축가와 시공사의 공유 도면

> 디자이너와 개발자가 **같은 도면**을 보며 일하는 세계

---

## 비유: 건축가와 시공사

집을 짓는다고 상상해보세요.

```
🏗️ 전통적인 방식 (Figma MCP 없이)
─────────────────────────────────────
건축가 (디자이너)          시공사 (개발자)
  │                          │
  ├── 도면을 그림              │
  ├── PDF로 출력 📄           │
  ├── 이메일로 전달 ────────→  ├── PDF를 받음
  │                          ├── 눈으로 보고 해석
  │                          ├── 직접 측정해서 코딩
  │   "이거 3px인가 4px인가?"  ├── 추측 😰
  │                          └── 완성... 했는데 다르다?!
  │
  └── "이거 내가 의도한 게 아닌데..."
```

```
🚀 Figma MCP 방식
─────────────────────────────────────
건축가 (디자이너)          시공사 (개발자 + Claude)
  │                          │
  ├── Figma에서 디자인 ◀──────┤── Figma 도면을 직접 읽음
  │   (살아있는 도면)          ├── 정확한 값을 자동 추출
  │                          ├── 코드로 변환
  │                          ├── 코드를 다시 Figma로! ──→ 확인
  │                          └── 완성! 정확히 일치 ✅
  └── "완벽하네!"
```

핵심은 **살아있는 공유 도면**입니다. PDF로 전달하는 게 아니라, 디자이너와 개발자가 같은 Figma 파일을 **실시간으로** 보고, 읽고, 수정합니다.

---

## Figma MCP가 할 수 있는 것

| 기능 | 설명 | 비유 |
|------|------|------|
| **디자인 읽기** | Figma 파일의 구조, 색상, 크기 등을 읽어옴 | 도면을 자동으로 읽는 로봇 |
| **코드 → 캔버스** | HTML/CSS 코드를 Figma 디자인으로 변환 | 시공 결과를 도면 위에 올려보기 |
| **디자인 토큰 추출** | 색상, 폰트, 간격 등 디자인 시스템 값 추출 | 도면의 규격표를 자동 정리 |
| **스타일 비교** | 코드와 디자인의 차이점 분석 | 시공 결과 vs 도면 대조 검사 |

## 양방향 워크플로우 (Round-trip)

Figma MCP의 가장 강력한 기능은 **양방향**이라는 점입니다:

```
┌─────────────┐     읽기      ┌──────────────┐
│             │ ──────────→  │              │
│   Figma     │              │  Claude Code │
│  (디자인)    │ ←────────── │   (코드)      │
│             │   쓰기       │              │
└─────────────┘  (Canvas)    └──────────────┘
       ↕                            ↕
   디자이너가                    개발자가
   디자인 수정                   코드 수정

       └────── 항상 동기화! ──────┘
```

1. **Figma → Code**: 디자인을 읽어서 코드로 변환
2. **Code → Figma**: 코드를 Figma 캔버스에 렌더링 (Code to Canvas)
3. **수정 → 다시 동기화**: 어느 쪽이 수정하든 다시 맞출 수 있음

## MCP 커넥터와의 관계

[11-mcp-connectors](../../11-mcp-connectors/)에서 배운 것처럼, Figma MCP도 **리모트 MCP 서버**의 하나입니다:

| 비교 | Slack 커넥터 | Figma MCP |
|------|-------------|-----------|
| 연결 대상 | Slack (메시지) | Figma (디자인) |
| 인증 방식 | OAuth | OAuth (Figma 계정) |
| 주요 도구 | 메시지 읽기/보내기 | 디자인 읽기/캔버스 쓰기 |
| 사용 장소 | claude.ai + Claude Code | claude.ai + Claude Code |

> Figma MCP는 11편에서 배운 커넥터 개념의 **확장판**이에요.
> "읽기만 가능한" 커넥터에서 "읽기 + 쓰기"가 가능한 양방향 도구로 진화한 겁니다!
