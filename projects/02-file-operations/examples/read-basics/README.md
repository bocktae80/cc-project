# 파일 읽기 기초 ⭐

> 클로드에게 "이 파일 읽어줘"라고 하면 어떻게 될까? 📖

## Read 도구란?

**Read 도구**는 클로드 코드가 파일의 내용을 읽어보는 도구예요.
마치 도서관에서 사서에게 **"이 책 좀 읽어주세요"**라고 부탁하는 것과 같아요!

```
📚 도서관 비유:
  나: "3번 서가에 있는 '수학의 정석' 읽어주세요"
  사서: (책을 꺼내서) "1페이지부터 읽어드릴게요..."

💻 클로드 코드:
  나: "sample.txt 파일 읽어줘"
  클로드: (Read 도구 사용) "파일 내용은 이렇습니다..."
```

✅ **안전한 도구!** — 읽기만 하니까 파일이 바뀌거나 삭제될 걱정이 없어요.

---

## 기본 사용법

### 1. 파일 전체 읽기

클로드에게 이렇게 말해보세요:

```
"sample.txt 파일 읽어줘"
```

클로드가 Read 도구를 사용해서 파일 내용을 보여줍니다.

### 2. 일부분만 읽기 (limit)

파일이 너무 길면 일부분만 읽을 수 있어요:

```
"sample.txt 파일의 처음 3줄만 읽어줘"
```

→ 클로드가 `limit: 3` 옵션을 사용합니다.

### 3. 중간부터 읽기 (offset)

파일의 중간부터 읽고 싶다면:

```
"sample.txt의 5번째 줄부터 3줄 읽어줘"
```

→ 클로드가 `offset: 5`, `limit: 3` 옵션을 사용합니다.

## 파라미터 정리 📋

| 파라미터 | 필수? | 설명 | 기본값 |
|----------|-------|------|--------|
| `file_path` | ✅ 필수 | 파일의 **절대 경로** | - |
| `limit` | 선택 | 읽을 줄 수 | 전체 (최대 2000줄) |
| `offset` | 선택 | 시작 줄 번호 | 1 (처음부터) |

💡 **절대 경로란?** 파일의 전체 주소예요.
- ✅ `/Users/kent/Work/project/sample.txt` (절대 경로)
- ❌ `sample.txt` (상대 경로 — Read 도구에서는 사용 불가!)

## 따라하기: 다양한 파일 읽어보기 🏃

### 실습 1: 텍스트 파일 읽기

이 폴더에 있는 `sample.txt`를 읽어보세요!

```
"sample.txt 읽어줘"
```

📌 **확인할 점:** 줄 번호가 함께 표시되는 것을 볼 수 있어요.

### 실습 2: JavaScript 파일 읽기

```
"sample.js 읽어줘"
```

📌 **확인할 점:** 코드에 구문 강조(syntax highlighting)가 적용되어 보여요.

### 실습 3: JSON 파일 읽기

```
"sample-data.json 읽어줘"
```

📌 **확인할 점:** JSON 구조가 들여쓰기와 함께 깔끔하게 표시돼요.

### 실습 4: 부분 읽기

```
"sample.txt의 3번째 줄부터 5줄만 읽어줘"
```

📌 **확인할 점:** 3~7번째 줄만 표시되는지 확인해보세요!

## 핵심 정리 ✨

```
📖 Read 도구 = 파일 읽기 전문가

✅ 기억할 것:
  1. file_path는 반드시 절대 경로!
  2. limit으로 읽을 줄 수 제한 가능
  3. offset으로 시작 위치 지정 가능
  4. 읽기만 하니까 100% 안전!
  5. txt, js, json 등 모든 텍스트 파일 읽기 가능
```
