# 파일 만들기 ⭐

> 클로드에게 "새 파일 만들어줘"라고 하면? ✨

## Write 도구란?

**Write 도구**는 클로드 코드가 새로운 파일을 만들거나, 기존 파일을 통째로 교체하는 도구예요.
마치 **새 노트에 글을 쓰는 것**과 같아요!

```
📚 도서관 비유:
  나: "이 내용으로 새 책을 만들어서 서가에 꽂아주세요"
  사서: "네, 새 책을 만들었습니다!"

⚠️ 같은 제목의 책이 이미 있다면?
  사서: "기존 책을 빼고 새 책으로 교체했습니다!"
        (기존 내용은 사라짐!)
```

---

## ⚠️ 가장 중요한 경고: 덮어쓰기!

```
🚨🚨🚨 Write 도구의 최대 주의사항! 🚨🚨🚨

같은 이름의 파일이 이미 있으면 → 기존 내용이 전부 사라져요!

예시:
  diary.txt (원래 내용: "오늘 학교에서 재미있었다...")
    ↓ Write 도구로 새 내용 쓰기
  diary.txt (새 내용: "내일 소풍이다!")
    → 원래 내용 "오늘 학교에서..." 는 영원히 사라짐! 😱
```

💡 **안전 규칙:**
- 새 파일을 만들 때 → Write ✅
- 기존 파일의 일부만 고칠 때 → Edit 사용! (Write ❌)

---

## 기본 사용법

### 1. 새 파일 만들기

클로드에게 이렇게 말해보세요:

```
"hello.txt 파일을 만들어줘. 내용은 '안녕하세요!'"
```

→ 클로드가 Write 도구를 사용해서 파일을 생성합니다.

### 2. 코드 파일 만들기

```
"인사하는 JavaScript 프로그램을 greeting.js로 만들어줘"
```

→ 클로드가 코드를 작성해서 파일로 저장합니다.

### 3. 여러 파일 한 번에 만들기

```
"index.html, style.css, app.js 3개 파일을 만들어줘"
```

→ 클로드가 여러 Write 도구를 동시에 사용합니다!

## 파라미터 정리 📋

| 파라미터 | 필수? | 설명 | 예시 |
|----------|-------|------|------|
| `file_path` | ✅ 필수 | 만들 파일의 절대 경로 | `/Users/me/project/hello.txt` |
| `content` | ✅ 필수 | 파일에 쓸 내용 | `"Hello World!"` |

📌 **참고:**
- 폴더가 없으면 자동으로 생성해줘요
- 파일 확장자에 맞게 내용을 작성하면 돼요 (`.js`면 JavaScript, `.html`이면 HTML)

---

## Write vs Edit 다시 한번! 🔄

이 차이는 정말 중요해서 한번 더 정리할게요:

| 상황 | 올바른 도구 | 이유 |
|------|------------|------|
| 새 파일 만들기 | **Write** ✅ | 아직 없는 파일이니까 |
| 파일 전체 다시 쓰기 | **Write** ⚠️ | 기존 내용이 사라져도 괜찮을 때만! |
| 한 줄만 고치기 | **Edit** ✅ | 나머지 내용은 그대로 유지 |
| 변수 이름 바꾸기 | **Edit** ✅ | 해당 부분만 교체 |
| 빈 파일에 내용 넣기 | **Write** ✅ | 잃을 내용이 없으니까 |

```
쉽게 외우기:
  Write = 새 종이에 쓰기 📄 (기존 종이는 버림!)
  Edit  = 지우개로 지우고 다시 쓰기 ✏️ (나머지는 그대로!)
```

## 핵심 정리 ✨

```
📝 Write 도구 = 파일 생성 전문가

✅ 기억할 것:
  1. file_path와 content 2개가 필수!
  2. 기존 파일이 있으면 덮어쓰기 됨! (주의!)
  3. 새 파일 만들 때 가장 유용
  4. 기존 파일 일부 수정은 Edit을 사용할 것!
  5. 폴더가 없으면 자동 생성됨
```
