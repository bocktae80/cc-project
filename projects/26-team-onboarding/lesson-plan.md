# 튜터 지시서: /team-onboarding — 팀 온보딩 가이드 생성

> 이 파일은 `/learn 26` 실행 시 튜터(클로드)가 읽는 지시서입니다. 사용자에게 직접 보여주지 마세요.

## 학습 목표

`/team-onboarding` 커맨드로 팀원이 Claude Code에 빠르게 적응할 수 있는 온보딩 가이드를 자동 생성하는 방법을 이해한다.

## 핵심 비유

"신입 사원 환영 키트를 자동으로 만들어주는 HR 담당자! 지난 30일간 당신이 어떻게 Claude Code를 썼는지 분석하고, 새 팀원이 따라할 수 있는 맞춤 가이드를 만들어줘요."

## 교육 흐름

### Step 1: /team-onboarding이란? (HR 환영 키트 비유)

1. "새 팀원이 들어왔을 때 '클로드 코드 어떻게 써요?'라는 질문에 답하기 어렵죠? /team-onboarding이 자동으로 가이드를 만들어줘요!"
2. HR 환영 키트 비유:
   - 팀이 어떤 일에 클로드를 쓰는지 (업무 유형 분석)
   - 자주 쓰는 도구가 뭔지 (커맨드/MCP 통계)
   - 어떤 걸 준비해야 하는지 (셋업 체크리스트)
3. 30일간 사용 데이터를 자동 분석

**확인 질문**: "새 팀원이 Claude Code를 시작할 때 가장 먼저 알아야 할 것은?"
→ 기대 답변: 팀이 어떤 작업에 어떻게 사용하는지, 필요한 셋업이 뭔지

### Step 2: 가이드 구조 이해

1. **업무 유형 비중**: Build Feature, Debug, Plan 등 비율
2. **Top 커맨드/MCP**: 팀이 자주 쓰는 슬래시 커맨드와 MCP 서버
3. **셋업 체크리스트**: 레포, MCP 서버, 스킬 목록
4. **팀 팁 & 스타터 태스크**: 온보딩용 커스텀 섹션

### Step 3: 실습 — /team-onboarding 실행

1. `/team-onboarding` 입력
2. 자동 분석 결과 확인
3. ONBOARDING.md 파일 생성됨

### Step 4: 가이드 커스터마이즈

1. 팀 이름 수정
2. 팀 팁 추가 (CLAUDE.md에 없는 노하우)
3. 스타터 태스크 설정

### Step 5: 새 팀원의 사용법

1. 새 팀원이 ONBOARDING.md를 Claude Code에 붙여넣기
2. Claude가 온보딩 버디로 변신 — 대화형 안내 시작
3. 셋업 체크리스트 하나씩 진행
