# Step 1: 커넥터 활성화하기

> claude.ai에서 클릭 몇 번으로 외부 서비스를 연결합니다.

---

## 전체 과정 미리보기

```
claude.ai 접속 → 설정 → MCP Connectors → 서비스 선택 → OAuth 허용 → 완료!
```

아래에서 하나씩 따라가 봅시다.

---

## 1단계: claude.ai 접속

웹 브라우저에서 [claude.ai](https://claude.ai)에 접속하고 로그인하세요.

```
[스크린샷 위치: claude.ai 메인 화면]
- 좌측 사이드바가 보이는 메인 화면
- 로그인된 상태
```

---

## 2단계: 설정 페이지 이동

프로필 아이콘 또는 설정 메뉴를 클릭합니다.

```
[스크린샷 위치: 설정 메뉴 접근]

경로: 프로필 아이콘 → Settings
```

> 설정 페이지에 도착하면 여러 탭이 보입니다.
> **Integrations** 또는 **MCP Connectors** 항목을 찾으세요.

---

## 3단계: MCP Connectors 찾기

설정 페이지에서 MCP 관련 섹션을 찾습니다.

```
[스크린샷 위치: 설정 페이지의 MCP Connectors 섹션]

Settings
├── Profile
├── Appearance
├── ...
└── Integrations / MCP Connectors   ← 여기!
```

이 섹션에 사용 가능한 커넥터 목록이 표시됩니다:

```
MCP Connectors
──────────────────────────
[ ] Slack         Slack 워크스페이스에 연결합니다
[ ] Notion        Notion 워크스페이스에 연결합니다
...
```

---

## 4단계: Slack 커넥터 활성화

### 활성화

Slack 커넥터 옆의 토글을 켜거나 "Connect" 버튼을 클릭합니다.

```
[스크린샷 위치: Slack 커넥터 활성화]

MCP Connectors
──────────────────────────
[x] Slack         ← 토글 ON 또는 "Connect" 클릭
```

### OAuth 인증 팝업

버튼을 클릭하면 **Slack 로그인 팝업**이 나타납니다.

```
[스크린샷 위치: Slack OAuth 인증 팝업]

┌─────────────────────────────────┐
│       Slack에 로그인             │
│                                  │
│  워크스페이스: myteam.slack.com  │
│                                  │
│  Claude가 다음을 요청합니다:     │
│  - 채널 메시지 읽기              │
│  - 메시지 보내기                 │
│  - 채널 목록 보기                │
│                                  │
│  [거부]         [허용]           │
└─────────────────────────────────┘
```

**"허용"을 클릭하세요.**

> 이것이 OAuth입니다! 비밀번호를 알려주지 않고, "이 기능만 허용"하는 거예요.
> 나중에 언제든지 허용을 취소할 수 있습니다.

### 연결 완료 확인

허용 후 설정 페이지로 돌아오면 상태가 "Connected"로 바뀝니다.

```
MCP Connectors
──────────────────────────
[x] Slack         Connected   [Disconnect]
```

## 5단계: Notion 커넥터 활성화

Slack과 같은 과정입니다:

1. Notion 커넥터 "Connect" 클릭
2. Notion 로그인 팝업
3. 접근 허용할 페이지/워크스페이스 선택
4. "허용" 클릭
5. 연결 완료

```
[스크린샷 위치: Notion OAuth 인증 팝업]

┌─────────────────────────────────┐
│    Notion에 접근 허용             │
│                                  │
│  Claude가 접근할 페이지 선택:     │
│  [x] 모든 페이지                 │
│  [ ] 특정 페이지만               │
│                                  │
│  [취소]         [접근 허용]       │
└─────────────────────────────────┘
```

---

## 6단계: Claude Code에서 확인

이제 Claude Code를 열고 커넥터가 잘 연결되었는지 확인해봅시다.

### 방법 1: 직접 물어보기

```
사용자: "Slack에 연결되어 있어?"
```

클로드가 ToolSearch를 통해 Slack 관련 도구가 있는지 확인하고 알려줄 거예요.

### 방법 2: ToolSearch 사용

클로드에게 도구를 검색해달라고 요청합니다:

```
사용자: "사용 가능한 Slack 도구가 뭐가 있어?"
```

연결이 잘 되었다면, 이런 도구들이 보일 거예요:

```
사용 가능한 Slack 도구:
- slack_read_channel       채널 메시지 읽기
- slack_send_message       메시지 보내기
- slack_search_public      메시지 검색
- slack_search_channels    채널 검색
- ...
```

## 문제 해결

### "커넥터가 안 보여요"

| 증상 | 해결 방법 |
|------|----------|
| 설정에 MCP Connectors 항목이 없음 | Claude Code 버전 확인 (v2.1.46 이상 필요) |
| 커넥터 목록이 비어있음 | 조직/플랜에 따라 사용 가능 여부가 다를 수 있음 |
| OAuth 팝업이 안 뜸 | 팝업 차단기 확인, 브라우저 설정 확인 |

### "연결했는데 Claude Code에서 안 돼요"

| 증상 | 해결 방법 |
|------|----------|
| 도구가 검색 안 됨 | Claude Code 재시작 |
| "권한 없음" 오류 | OAuth 재인증 (Disconnect → 다시 Connect) |
| 특정 채널/페이지 접근 불가 | 해당 서비스에서 권한 확인 |

---

## 핵심 정리

```
1. claude.ai 설정에서 커넥터를 활성화
2. OAuth 팝업에서 "허용" 클릭
3. Claude Code에서 바로 사용 가능
4. API 키나 토큰을 직접 관리할 필요 없음!
```
