# Slack 커넥터 실습

> Slack 채널을 읽고, 메시지를 검색하고, 메시지를 보내봅시다!

---

## 사전 준비

- [Step 1](../../tutorial/step-01-setup.md)에서 Slack 커넥터를 활성화한 상태
- Slack 워크스페이스에 접근 가능한 상태

---

## 실습 1: 채널 메시지 읽기

### 해보기

Claude Code에서 아래처럼 요청하세요:

```
"#general 채널의 최근 메시지 5개 보여줘"
```

### 어떤 일이 일어나나

```
사용자 요청
    |
    v
ToolSearch("slack read channel")
    |
    v
slack_read_channel 도구 발견 및 로딩
    |
    v
slack_read_channel(channel: "general") 호출
    |
    v
Slack API에서 메시지 가져옴
    |
    v
사용자에게 정리된 결과 전달
```

### 예상 결과

```
#general 채널의 최근 메시지입니다:

1. 김철수 (2월 22일 09:30)
   "오늘 스프린트 리뷰 있습니다"

2. 이영희 (2월 22일 10:15)
   "리뷰 자료 공유합니다 [링크]"

3. 박민수 (2월 22일 11:00)
   "넵, 확인했습니다!"
...
```

### 다양한 요청 방법

```
"#dev 채널 읽어줘"
"#random 채널에 오늘 올라온 메시지 보여줘"
"프론트엔드 채널의 최근 대화 보여줘"
```

## 실습 2: 메시지 검색하기

### 해보기

```
"Slack에서 '회의' 관련 메시지를 찾아줘"
```

### 예상 결과

```
'회의' 관련 메시지를 검색했습니다:

1. #general - 김팀장 (2월 21일)
   "내일 팀 회의 10시에 합시다"

2. #dev - 이개발 (2월 20일)
   "기술 회의 안건 공유합니다"

3. #pm - 박기획 (2월 19일)
   "주간 회의록 정리했습니다"
```

### 검색 범위 지정

```
"#dev 채널에서만 '배포' 검색해줘"
"이번 주 메시지 중에서 '버그' 찾아줘"
"김철수가 보낸 메시지 중에서 '리뷰' 찾아줘"
```

## 실습 3: 메시지 보내기

### 해보기

```
"#general에 '오늘 점심 메뉴 추천받습니다!' 메시지 보내줘"
```

### 확인 절차

클로드는 메시지를 보내기 전에 확인을 요청합니다:

```
클로드: "다음 메시지를 #general에 보내겠습니다:
        '오늘 점심 메뉴 추천받습니다!'
        보내도 될까요?"

사용자: "응, 보내줘"

클로드: "메시지를 전송했습니다!"
```

> 메시지 전송은 되돌릴 수 없으니, 채널과 내용을 꼭 확인하세요.

### 다양한 전송 방법

```
"#dev에 '빌드 완료되었습니다' 보내줘"
"#random에 재미있는 인사 보내줘"
```

## 실습 4: 채널 검색하기

### 해보기

```
"'프론트엔드' 관련 Slack 채널 찾아줘"
```

### 예상 결과

```
1. #frontend - 프론트엔드 팀 채널 (멤버 12명)
2. #frontend-review - 코드 리뷰 채널 (멤버 8명)
3. #frontend-study - 스터디 채널 (멤버 5명)
```

## 실습 5: 스레드 읽기

### 해보기

특정 메시지에 달린 답글(스레드)을 확인할 수 있어요.

```
"그 메시지에 달린 답글(스레드) 보여줘"
```

### 예상 결과

```
스레드 내용 (답글 3개):

원본: 김팀장 - "내일 회의 10시에 합시다"
├── 이영희: "네, 참석합니다"
├── 박민수: "저도 참석이요"
└── 최지은: "10시 30분으로 조정 가능할까요?"
```

## 자주 쓰는 시나리오

### 업무 시작할 때

```
"내가 멘션된 Slack 메시지 찾아줘"
"#dev 채널에 어제 올라온 메시지 요약해줘"
```

### 정보 찾을 때

```
"Slack에서 'API 문서' 검색해줘"
"'에러' 관련 최근 메시지 찾아줘"
```

### 소통할 때

```
"#general에 공지사항 보내줘"
"테스트 완료 메시지를 #qa에 보내줘"
```

---

## 주의 사항

| 항목 | 설명 |
|------|------|
| 접근 범위 | OAuth에서 허용한 채널만 접근 가능 |
| DM | 개인 메시지(DM)는 보통 접근 불가 |
| 전송 주의 | 보내기 전에 채널/내용 꼭 확인 |
| 검색 한계 | 매우 오래된 메시지는 검색이 안 될 수 있음 |

---

## 핵심 정리

```
채널 읽기: "#{채널명} 메시지 보여줘"
메시지 검색: "Slack에서 '{키워드}' 찾아줘"
메시지 보내기: "#{채널명}에 '{내용}' 보내줘"
채널 검색: "'{키워드}' 관련 채널 찾아줘"
스레드 읽기: "그 메시지의 답글 보여줘"
```
