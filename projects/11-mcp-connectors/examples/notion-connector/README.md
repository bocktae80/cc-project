# Notion 커넥터 실습

> Notion 페이지를 검색하고, 내용을 읽고, 댓글을 달아봅시다!

---

## 사전 준비

- [Step 1](../../tutorial/step-01-setup.md)에서 Notion 커넥터를 활성화한 상태
- Notion 워크스페이스에 접근 가능한 상태
- 실습용 페이지가 하나 이상 있으면 좋아요 (없어도 괜찮습니다)

---

## 실습 1: 페이지 검색하기

### 해보기

Claude Code에서 아래처럼 요청하세요:

```
"Notion에서 '회의록' 페이지 찾아줘"
```

### 어떤 일이 일어나나

```
사용자 요청
    |
    v
ToolSearch("notion search")
    |
    v
notion-search 도구 발견 및 로딩
    |
    v
notion-search(query: "회의록") 호출
    |
    v
Notion API에서 페이지 검색
    |
    v
사용자에게 검색 결과 전달
```

### 예상 결과

```
'회의록' 검색 결과:

1. "2026년 2월 주간 회의록"
   마지막 수정: 2월 21일
   위치: 팀 워크스페이스 > 회의

2. "프로젝트 킥오프 회의록"
   마지막 수정: 2월 15일
   위치: 프로젝트 A

3. "회의록 템플릿"
   마지막 수정: 1월 10일
   위치: 템플릿
```

### 다양한 검색 방법

```
"Notion에서 '프로젝트 계획' 검색해줘"
"'디자인 시안' 관련 Notion 페이지 찾아줘"
"최근 수정된 Notion 문서 보여줘"
```

## 실습 2: 페이지 내용 읽기

### 해보기

검색 결과에서 특정 페이지의 내용을 읽어봅시다:

```
"그 중에서 첫 번째 페이지 내용 보여줘"
```

또는 직접 지정:

```
"Notion에서 '2026년 2월 주간 회의록' 내용 보여줘"
```

### 예상 결과

```
"2026년 2월 주간 회의록" 내용:

# 2026년 2월 주간 회의록

## 참석자
- 김팀장, 이영희, 박민수, 최지은

## 안건
1. 이번 주 진행 상황 공유
2. 다음 주 계획
3. 이슈 논의

## 결정 사항
- API 리팩토링은 다음 스프린트로 연기
- 디자인 리뷰는 수요일 진행
...
```

### 활용 팁

```
"이 페이지 내용을 3줄로 요약해줘"
"이 페이지에서 결정 사항만 뽑아줘"
"이 문서에 언급된 일정을 표로 정리해줘"
```

> Notion 페이지 내용을 읽은 후 클로드에게 분석/요약을 요청할 수 있어요!

## 실습 3: 댓글 달기

### 해보기

페이지에 댓글을 추가해봅시다:

```
"그 페이지에 '리뷰 완료. 3번 안건은 추가 논의 필요' 댓글 달아줘"
```

### 확인 절차

```
클로드: "'2026년 2월 주간 회의록' 페이지에 댓글을 추가하겠습니다:
        '리뷰 완료. 3번 안건은 추가 논의 필요'
        추가해도 될까요?"

사용자: "응"

클로드: "댓글을 추가했습니다!"
```

### 기존 댓글 확인

```
"이 페이지에 달린 댓글 보여줘"
```

```
댓글 목록:

1. 이영희 (2월 21일): "API 리팩토링 관련 문서 링크 추가했습니다"
2. 박민수 (2월 22일): "디자인 리뷰 수요일 2시로 확정합니다"
3. 나 (방금): "리뷰 완료. 3번 안건은 추가 논의 필요"
```

## 자주 쓰는 시나리오

### 문서 찾기 + 요약

```
"Notion에서 'API 문서' 찾아서 핵심 내용만 요약해줘"
```

클로드가 검색 → 내용 읽기 → 요약까지 한번에 처리합니다.

### 회의 준비

```
"Notion에서 지난주 회의록 찾아서 결정 사항 정리해줘.
 이번 주 해야 할 일 목록도 만들어줘."
```

### 문서 리뷰

```
"Notion에서 '기획서' 페이지 읽고,
 빠진 부분이나 보완할 점 알려줘.
 피드백을 댓글로 달아줘."
```

### Slack과 조합

```
"Notion에서 이번 주 할 일 페이지 찾아서 내용 읽어줘.
 그리고 그 내용을 요약해서 Slack #general에 공유해줘."
```

> MCP 커넥터의 진짜 힘은 **여러 서비스를 조합**할 때 나타납니다!

## 주의 사항

| 항목 | 설명 |
|------|------|
| 접근 범위 | OAuth에서 허용한 페이지/워크스페이스만 접근 가능 |
| 비공개 페이지 | 다른 사람의 비공개 페이지는 접근 불가 |
| 수정 권한 | 댓글은 달 수 있지만, 페이지 내용 수정은 제한적일 수 있음 |
| 대용량 페이지 | 매우 긴 페이지는 일부만 가져올 수 있음 |

## 사용 가능한 도구 정리

| 도구 | 기능 | 사용 예 |
|------|------|---------|
| `notion-search` | 검색 | "'프로젝트' 페이지 찾아줘" |
| `notion-fetch` | 내용 읽기 | "이 페이지 내용 보여줘" |
| `notion-create-pages` | 페이지 만들기 | "새 페이지 만들어줘" |
| `notion-create-comment` | 댓글 달기 | "이 페이지에 댓글 달아줘" |
| `notion-create-database` | DB 만들기 | "새 데이터베이스 만들어줘" |
| `notion-update-page` | 페이지 수정 | "이 페이지 제목 바꿔줘" |
| `notion-get-comments` | 댓글 보기 | "댓글 목록 보여줘" |
| `notion-get-users` | 멤버 보기 | "워크스페이스 멤버 보여줘" |
| `notion-duplicate-page` | 페이지 복제 | "이 페이지 복사해줘" |
| `notion-move-pages` | 페이지 이동 | "이 페이지를 다른 곳으로 옮겨줘" |

---

## 핵심 정리

```
페이지 검색: "Notion에서 '{키워드}' 찾아줘"
내용 읽기: "이 페이지 내용 보여줘"
댓글 달기: "이 페이지에 '{내용}' 댓글 달아줘"
댓글 보기: "이 페이지 댓글 보여줘"
조합 활용: Notion + Slack으로 정보 공유!
```
