# MCP 커넥터란?

> 학교 공유 프린터처럼, 누군가 설치해둔 도구를 그냥 쓰는 것!

---

## 공유 프린터 비유

여러분 학교에 **공유 프린터**가 있다고 생각해보세요.

```
🏫 학교 공유 프린터 시스템
────────────────────────────────────────────

[관리자(선생님)]                    [학생들]
   |                                  |
   |  1. 프린터 구매                  |
   |  2. 네트워크 연결                |
   |  3. 드라이버 설치                |
   |  4. 권한 설정                    |
   |                                  |
   v                                  v
+-----------+                  +-----------+
| 프린터    | <--- 네트워크 ---| 학생 PC   |
| (설정 완료)|                  | (인쇄만!) |
+-----------+                  +-----------+
```

- **선생님(관리자)**: 프린터를 사서, 네트워크에 연결하고, 드라이버를 설치합니다
- **학생(사용자)**: "인쇄" 버튼만 누르면 됩니다. 프린터가 어떻게 설치됐는지 몰라도 OK!

**MCP 커넥터도 완전히 같습니다!**

```
MCP 커넥터 시스템
────────────────────────────────────────────

[관리자(Anthropic)]                [사용자(나)]
   |                                  |
   |  1. MCP 서버 구축                |
   |  2. OAuth 인증 연결              |
   |  3. 보안 설정                    |
   |  4. API 연동                     |
   |                                  |
   v                                  v
+-----------+                  +-----------+
| 커넥터    | <--- claude.ai --| Claude    |
| (설정 완료)|                  | Code      |
+-----------+                  +-----------+
```

- **Anthropic(관리자)**: MCP 서버를 구축하고, 인증을 설정하고, 보안을 관리합니다
- **나(사용자)**: claude.ai에서 "활성화" 버튼만 누르면 됩니다!

## 07에서 배운 것 vs 이번에 배울 것

[07-mcp-server](../../07-mcp-server/)에서는 **직접** MCP 서버를 설정했죠:

```
07-mcp-server (직접 설정)
──────────────────────────
"나"가 프린터를 사서 → 드라이버 설치 → Wi-Fi 연결 → 용지/잉크 관리

  나 → claude mcp add → 서버 이름/명령어 지정 → 환경변수 설정 → 완료
```

이번 11-mcp-connectors에서는 **이미 설정된 것**을 사용합니다:

```
11-mcp-connectors (커넥터)
──────────────────────────
"Anthropic"이 프린터 설치 → 네트워크 연결 → 모두 사용 가능

  나 → claude.ai 설정 → 커넥터 활성화 → OAuth "허용" 클릭 → 완료!
```

## 사용 가능한 커넥터

현재 제공되는 주요 MCP 커넥터입니다:

| 커넥터 | 할 수 있는 일 | 필요한 것 |
|--------|--------------|----------|
| **Slack** | 채널 읽기, 메시지 검색, 메시지 보내기 | Slack 워크스페이스 |
| **Notion** | 페이지 검색, 내용 읽기, 댓글 작성 | Notion 워크스페이스 |

> 커넥터 목록은 계속 추가되고 있어요. claude.ai 설정 페이지에서 최신 목록을 확인하세요.

## OAuth — API 키가 필요 없어요

07에서는 GitHub 서버를 연결할 때 **Personal Access Token**을 직접 발급받아야 했죠?

```
07-mcp-server의 인증 방식
──────────────────────────
1. GitHub 설정 → Developer Settings → Personal Access Token
2. 토큰 복사
3. 환경변수에 GITHUB_TOKEN=ghp_xxxx... 설정
4. 토큰 만료되면? 다시 발급...
```

커넥터는 **OAuth**를 사용합니다:

```
MCP 커넥터의 인증 방식 (OAuth)
──────────────────────────
1. claude.ai에서 "Slack 연결" 클릭
2. Slack 로그인 팝업 나타남
3. "허용" 클릭
4. 끝! (토큰 관리는 Anthropic이 알아서)
```

### OAuth란?

**O**pen **Auth**orization의 줄임말이에요. 쉽게 말하면:

> "내 비밀번호를 알려주지 않고도, 다른 서비스가 내 데이터에 접근하도록 허용하는 방법"

비유하면 **호텔 카드키**와 비슷해요:

```
호텔 카드키                           OAuth 토큰
──────────                           ──────────
프론트에서 신원 확인                   로그인으로 본인 확인
카드키 발급                           토큰 발급
내 방만 열 수 있음                    허용한 데이터만 접근
체크아웃하면 무효                      연결 끊으면 무효
마스터키는 호텔만 가짐                 비밀번호는 서비스만 가짐
```

## 핵심 정리

```
MCP 커넥터 = claude.ai에서 제공하는 미리 설정된 MCP 서버
        = 학교 공유 프린터 (설치는 관리자가, 사용은 내가)
        = OAuth로 인증 자동 처리 (API 키 직접 관리 불필요)
        = Slack, Notion 등 인기 서비스 지원
```
