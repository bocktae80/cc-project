# 튜터 지시서: 브라우저 자동화 (Chrome)

> 이 파일은 `/learn 15` 실행 시 튜터(클로드)가 읽는 지시서입니다. 사용자에게 직접 보여주지 마세요.

## 학습 목표

Chrome 확장을 통해 클로드가 브라우저를 제어하는 원리를 이해하고, 페이지 탐색/스크린샷/GIF 녹화 등 실전 활용법을 배운다.

## 핵심 비유

"리모컨으로 TV를 조종하듯, 클로드가 브라우저를 조종해요! 채널 돌리기(페이지 이동), 볼륨 조절(폼 입력), 화면 캡처(스크린샷)까지 전부 가능해요."

## 연습 파일

- 없음 (브라우저 확장 설치 후 실습 위주)

## 교육 흐름

### Step 1: Chrome 확장과 연결 원리

1. "웹 브라우저도 클로드가 직접 조종할 수 있다는 거 알고 있었어요?"
2. 연결 구조 설명:
   - Chrome 확장(Extension) 설치 → CLI와 연결 → 클로드가 명령 전달
   - 확장이 브라우저 안에서 "통역관" 역할 (클로드의 명령 → 브라우저 동작)
3. 연결 흐름:
   - `claude` CLI 실행 → 확장과 WebSocket 연결 → 페이지 제어 가능
4. "TV 리모컨의 적외선 센서처럼, 확장이 클로드와 브라우저 사이를 연결해주는 거예요!"

### Step 2: 할 수 있는 것들

1. 주요 기능 소개:
   - **페이지 이동 (navigate)**: URL로 이동, 링크 클릭
   - **클릭 (click)**: 버튼, 링크, 메뉴 등 클릭
   - **폼 입력 (type)**: 텍스트 필드에 값 입력
   - **스크린샷 (screenshot)**: 현재 화면 캡처
   - **GIF 녹화**: 동작을 GIF로 기록
2. "리모컨으로 채널 돌리고, 음량 바꾸고, 녹화하는 것처럼 다양한 조종이 가능해요!"
3. 실습 제안: "간단한 페이지 이동부터 해볼까요?"

### Step 3: 디자인 검증 워크플로우

1. Figma(12)에서 배운 디자인과 연결:
   - Figma 디자인 → 코드 빌드 → 브라우저에서 열기 → 스크린샷 → 비교
2. 실전 워크플로우:
   - 디자인 시안과 실제 화면의 차이를 AI가 비교
   - 색상, 크기, 레이아웃 차이를 자동 발견
3. "건축에서 설계도와 실제 건물 사진을 비교하는 것처럼, 디자인과 실제 화면을 비교해요!"
4. 콘솔 로그, 네트워크 모니터링 활용도 언급

### Step 4: 마무리 퀴즈

퀴즈:
1. "Chrome 확장의 역할은?"
   → 정답: 클로드 CLI와 브라우저 사이를 연결하는 통역관 (WebSocket 연결)
2. "브라우저 자동화로 할 수 없는 것은?"
   → 정답: 브라우저 밖의 데스크톱 앱 조작 (브라우저 내에서만 동작)
3. "디자인 검증 워크플로우의 순서는?"
   → 정답: Figma 디자인 → 코드 빌드 → 브라우저 스크린샷 → 비교

### 마무리

"축하해요! 이제 클로드가 브라우저까지 조종할 수 있다는 걸 알게 됐어요!
리모컨 하나로 TV를 자유자재로 다루듯, 브라우저도 클로드에게 맡기면 돼요.
다음 추천: `/learn 16`으로 백그라운드 에이전트를 배워보세요!"
