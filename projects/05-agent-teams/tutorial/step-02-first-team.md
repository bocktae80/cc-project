# Step 2 — 첫 번째 팀 만들기 ⭐

> 팀을 만들고, 구조를 확인하고, 바로 삭제해보자! (비용 최소화)

---

## 목표

- 팀을 생성하는 방법을 배웁니다
- 팀 설정 파일(config.json)의 구조를 이해합니다
- 필요 없는 팀은 바로 삭제하는 습관을 들입니다

---

## 팀 만들기

Claude Code에서 이렇게 말해보세요:

```
hello-team이라는 팀을 만들어줘
```

그러면 Claude가 내부적으로 **TeamCreate** 도구를 실행합니다:

```
TeamCreate
├── name: "hello-team"
└── members: []  (아직 팀원 없음)
```

---

## 만들어진 파일 확인

팀이 만들어지면 설정 파일이 생성됩니다:

```bash
cat ~/.claude/teams/hello-team/config.json
```

**config.json 구조:**

```json
{
  "name": "hello-team",
  "members": [],
  "created_at": "2025-01-15T10:30:00Z"
}
```

| 필드 | 설명 |
|------|------|
| `name` | 팀 이름 (고유해야 함) |
| `members` | 팀원 목록 (에이전트 배열) |
| `created_at` | 생성 시각 |

> 팀원(`members`)은 나중에 에이전트를 추가하면 채워집니다. 지금은 빈 배열이에요.

---

## 바로 삭제하기

실습이 끝났으니 바로 삭제합시다. 비용을 아끼는 좋은 습관입니다!

```
hello-team을 삭제해줘
```

Claude가 **TeamDelete**를 실행합니다:

```
TeamDelete
└── name: "hello-team"
```

삭제 후 확인:

```bash
ls ~/.claude/teams/hello-team 2>/dev/null || echo "삭제 완료!"
```

---

## 왜 바로 삭제하나요?

```
팀 존재 = 에이전트 실행 가능 = 토큰 소모 가능
```

Agent Teams는 에이전트가 **각각 독립적으로 토큰을 사용**합니다. 불필요한 팀을 남겨두면 실수로 비용이 발생할 수 있어요. 연습할 때는 **만들고 → 확인하고 → 바로 삭제**하는 습관을 들이세요.

---

## 정리

| 명령 | 도구 | 역할 |
|------|------|------|
| "팀 만들어줘" | TeamCreate | 팀 생성 |
| "팀 삭제해줘" | TeamDelete | 팀 삭제 |
