# 튜터 지시서: 플러그인 시스템

> 이 파일은 `/learn 17` 실행 시 튜터(클로드)가 읽는 지시서입니다. 사용자에게 직접 보여주지 마세요.

## 학습 목표

플러그인 시스템의 개념을 이해하고, plugin.json 매니페스트 구조와 나만의 플러그인 만들기/배포 방법을 배운다.

## 핵심 비유

"스마트폰 앱스토어처럼, 필요한 기능을 설치해서 쓰기! 게임, 카메라, 계산기 앱을 따로 설치하듯, 클로드에도 필요한 기능 번들을 설치할 수 있어요."

## 연습 파일

- 없음 (개념 + 실습 위주)

## 교육 흐름

### Step 1: 플러그인이란?

1. "지금까지 Skills(08), Hooks(06), Agents(05)를 따로 배웠죠? 플러그인은 이것들을 하나로 묶는 거예요!"
2. 플러그인 구성 요소:
   - **Skills**: 슬래시 커맨드 (`/commit`, `/deploy`)
   - **Commands**: 자동 실행 명령
   - **Hooks**: 이벤트 반응 (저장 시 자동 실행 등)
   - **Agents**: 특수 목적 에이전트
3. 비유 활용:
   - 개별 앱 = Skills, Hooks, Agents (각각 설치)
   - 앱 번들 = 플러그인 (관련 기능을 한 번에 설치)
4. "게임기에 게임팩 하나 꽂으면 캐릭터, 맵, 음악이 전부 들어있는 것처럼!"

### Step 2: plugin.json 매니페스트 구조

1. plugin.json 파일 설명:
   - 플러그인의 "설명서" 역할
   - 이름, 버전, 포함된 기능 목록
   - 의존성 정보
2. 각 필드 설명:
   - `name`: 플러그인 이름
   - `version`: 버전 번호
   - `description`: 설명
   - `skills`: 포함된 스킬 목록
   - `hooks`: 포함된 훅 목록
   - `agents`: 포함된 에이전트 목록
3. "앱스토어에서 앱 정보 페이지를 보면 이름, 설명, 버전이 나오잖아요? plugin.json이 그 역할이에요!"

### Step 3: 나만의 플러그인 만들기 & 배포

1. 플러그인 만들기 과정:
   - plugin.json 작성
   - Skills/Hooks/Agents 파일 구성
   - 테스트
2. 배포 방법:
   - npm 패키지로 배포
   - Git 저장소로 공유
   - `claude plugin add <패키지명>` 으로 설치
3. "나만의 앱을 앱스토어에 올리는 것처럼, 내가 만든 플러그인을 다른 사람도 쓸 수 있어요!"

### Step 4: 마무리 퀴즈

퀴즈:
1. "플러그인이 포함할 수 있는 것들은?"
   → 정답: Skills, Commands, Hooks, Agents (관련 기능을 하나로 묶은 번들)
2. "plugin.json의 역할은?"
   → 정답: 플러그인의 이름, 버전, 포함된 기능 목록 등을 정의하는 설명서(매니페스트)
3. "플러그인을 설치하는 명령어는?"
   → 정답: claude plugin add <패키지명>

### 마무리

"축하해요! 이제 클로드 코드의 플러그인 시스템을 이해했어요!
앱스토어에서 앱을 설치하듯 기능을 추가하고, 나만의 플러그인도 만들어서 공유할 수 있어요.
지금까지 배운 Skills(08) + Hooks(06) + Agents(05)를 플러그인으로 묶어보는 것도 좋은 연습이에요!"
