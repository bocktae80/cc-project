# 24. /loop & Cron 스케줄링

> 반복 알람처럼, 클로드가 주기적으로 알아서 확인하고 보고하는 자동 감시 기능

**난이도:** ⭐⭐ (중급)
**선행 학습:** [10-cli-master](../10-cli-master/), [22-batch-mode](../22-batch-mode/)
**CLI 버전:** v2.1.71+

---

## 무엇을 배우나요?

- `/loop` 명령어로 프롬프트를 주기적으로 반복 실행하기
- `CronCreate`, `CronList`, `CronDelete` 도구로 세밀한 스케줄링
- 자연어 리마인더로 일회성 알림 설정
- 배포 감시, PR 모니터링, 서버 헬스체크 등 실전 활용

## 왜 배우나요?

지금까지 클로드에게 "이거 해줘"라고 한 번 말하면 한 번 실행하고 끝이었습니다.
하지만 개발할 때는 **계속 지켜봐야 하는 일**이 많습니다:

- 배포가 끝났는지 5분마다 확인
- PR에 새 코멘트가 달렸는지 체크
- 테스트가 통과했는지 기다리기

`/loop`은 이런 반복 확인을 자동화합니다. **알람 시계를 맞춰두면 비서가 주기적으로 확인하고 보고하는 것**과 같습니다.

## 핵심 비유

```
일반 대화:
  나: "배포 됐어?" → 클로드: "아직이요" → (5분 후) 나: "지금은?" → ...

/loop 사용:
  나: "/loop 2m 배포 됐는지 확인해줘"
  → 클로드가 2분마다 알아서 확인하고 보고
  → 나는 다른 일 하면 됨!
```

## 빠른 시작

```bash
# 1. 배포 완료 감시 (2분 간격)
/loop 2m check if the Vercel deployment finished

# 2. PR 코멘트 감시 (10분 간격)
/loop 10m check PR #42 for new review comments

# 3. 45분 후 리마인더 (일회성)
in 45 minutes, remind me to push the release branch

# 4. 예약 작업 목록 확인
what scheduled tasks do I have?

# 5. 작업 취소
cancel the deploy check job
```

## 주의사항

- **세션 스코프**: 터미널을 닫으면 모든 예약 작업이 사라집니다
- **3일 만료**: 반복 작업은 3일 후 자동 삭제됩니다
- **유휴 시 실행**: 클로드가 다른 응답 중이면 끝날 때까지 대기합니다
- 지속적 스케줄링이 필요하면 GitHub Actions나 Desktop 예약 작업을 사용하세요
