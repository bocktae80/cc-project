# 28 — /ultrareview

> 난이도: ⭐⭐⭐ (v2.1.111 신규)

클라우드에서 **여러 에이전트가 병렬로** 코드를 리뷰하고 서로 크리틱해서 통합 리포트를 돌려주는 새 리뷰 명령. 혼자 리뷰하면 놓치는 이슈를 다각도로 탐지합니다.

## 이런 상황에서 유용해요

- 대형 PR (500줄+) — 1인 리뷰 피로로 뒷부분을 놓치기 쉬움
- 보안/인증/권한 변경 — 여러 각도 검토가 필수
- 배포 직전 최종 점검 — 사람 리뷰 LGTM 후 한 번 더 크로스체크
- 리뷰어가 없을 때 AI 1차 리뷰 → 이슈 선제 수정 → 사람 2차 리뷰

## /review vs /ultrareview

| 항목 | /review | /ultrareview |
|------|---------|--------------|
| 실행 위치 | 로컬 세션 | 클라우드 |
| 에이전트 수 | 1개 | 여러 개 병렬 |
| 크리틱 루프 | 없음 | 있음 (에이전트 간) |
| 세션 블로킹 | 결과까지 대기 | 백그라운드 |
| PR 입력 | 현재 diff | 현재 브랜치 OR `<PR#>` |

## 빠른 시작

```bash
# 1) 현재 브랜치 리뷰
/ultrareview

# 2) 같은 레포의 PR 번호
/ultrareview 142

# 3) 다른 레포의 PR URL
/ultrareview https://github.com/org/repo/pull/142
```

## v2.1.113 개선

- 런치 자체가 **병렬화**되어 시작 속도 향상
- 런치 다이얼로그에 **diffstat 즉시 표시** → 잘못된 브랜치/너무 큰 PR 사전 감지
- 런치 중 **Esc로 취소** 가능 (크레딧 소비 전)
- "Launching..." **애니메이션 상태** 표시

## 주의사항

- `/ultrareview <PR#>`는 PR diff를 클라우드로 전송합니다. 민감한 private 저장소는 조직 정책을 확인하세요.
- 인자 없이(`/ultrareview`) 실행 시 현재 브랜치를 base 브랜치와 비교합니다 — base 브랜치 지정이 애매한 경우 `/ultrareview <PR#>`로 명시하는 게 안전합니다.
- 다른 레포의 PR을 리뷰하려면 `gh auth status`에서 해당 레포 읽기 권한이 있어야 합니다.
