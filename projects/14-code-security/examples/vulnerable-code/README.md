# 취약 코드 샘플

> 보안 취약점을 이해하기 위한 **교육용** 코드 모음

---

## 이 폴더는 뭔가요?

의도적으로 보안 취약점을 넣어둔 **교육용 코드 예제**입니다.

각 파일에는:
- **취약한 코드** (위험!) — 어떤 점이 문제인지 주석으로 설명
- **안전한 코드** (수정됨) — 어떻게 고쳤는지 주석으로 설명

```
절대 프로덕션(실제 서비스)에서 취약한 코드를 사용하지 마세요!
이 파일들은 오직 "이런 게 위험하다"를 이해하기 위한 교육 목적입니다.
```

## 학습 방법

### 방법 1: 직접 읽기

각 파일을 열어서 "취약한 코드"와 "안전한 코드"를 비교해보세요.

```
> 이 파일에서 어떤 부분이 위험한지 찾아보세요
> 힌트: 주석에 "취약한 코드 (위험!)"이라고 표시되어 있어요
```

### 방법 2: Claude에게 분석 요청

```
> sql-injection.js를 읽고, 취약한 부분을 찾아서 설명해줘

> xss-example.js의 취약한 코드와 안전한 코드의 차이점을 비교해줘

> broken-auth.js에서 공격자가 악용할 수 있는 시나리오를 설명해줘
```

### 방법 3: 문제 맞추기

```
> sql-injection.js의 취약한 코드만 보여줄게.
> 어떤 공격이 가능한지 맞춰볼래?
```

## 각 취약점 요약

### SQL 인젝션

```
위험: 사용자 입력이 DB 명령에 끼어들어 데이터 유출/삭제
해결: 파라미터화 쿼리 사용 (? 또는 $1 플레이스홀더)
기억: "사용자 입력을 절대 SQL에 직접 넣지 말 것"
```

### XSS

```
위험: 악성 스크립트가 다른 사용자의 브라우저에서 실행
해결: innerHTML 대신 textContent 사용, 또는 입력 이스케이프
기억: "사용자 입력을 절대 HTML로 직접 출력하지 말 것"
```

### 인증 우회

```
위험: 로그인 없이 보호된 리소스에 접근, 남의 정보 열람
해결: 인증 미들웨어 + 토큰에서 사용자 식별
기억: "모든 보호된 경로에 반드시 인증 확인"
```
