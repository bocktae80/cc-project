# 텔레포트란? — 순간이동 포탈

> 세션을 터미널에서 웹으로, 웹에서 터미널로 자유롭게 옮기는 마법

---

## 비유: 게임 포탈

게임에서 포탈을 열면 다른 장소로 순간이동하잖아요?

```
게임 세계 (기존 방식)                  포탈이 있는 세계 (텔레포트)
-----------------------------         -----------------------------

  [성 안에서 작업]                      [성 안에서 작업]
       |                                    |
       | 문 밖으로 나가려면                     | 포탈 열기!
       | 처음부터 다시 걸어가야 함                 |
       |                                    v
       v                              [마을로 순간이동]
  [길게 걸어가기...]                          |
       |                                    | 또 다른 포탈!
       v                                    v
  [마을 도착]                           [동굴로 순간이동]
                                            |
                                       즉시 이동 완료!
```

클로드 코드의 텔레포트도 마찬가지예요. **터미널(성)**에서 작업하다가, **포탈을 열면 웹(마을)**으로 순간이동! 또 다른 포탈을 열면 **다시 터미널로** 돌아올 수 있어요.

---

## 세 가지 핵심 기능

클로드 코드의 세션 이동에는 세 가지 방법이 있어요:

```
                        Claude Code 세션 이동
                    ============================

   1. & prefix               2. /teleport              3. /rc
   [보내기]                   [가져오기]                 [리모컨]

   Terminal                  Web (claude.ai)           Terminal
   +--------+               +--------+                +--------+
   |  내가   | --portal-->  |  웹에서 | --portal-->   |  내가   |
   |  작업중  |   클라우드    |  확인   |   터미널      |  작업중  |
   +--------+   로 보냄     +--------+   로 가져옴    +--------+
                                                          ^
                                                          |
                                                      +--------+
                                                      | 원격에서 |
                                                      | 조종!   |
                                                      +--------+
```

### 1. `&` prefix — 포탈로 보내기

프롬프트 앞에 `&`를 붙이면, 작업이 **클라우드에서 실행**됩니다.

```bash
# 터미널에서 이렇게 입력하면:
& 이 프로젝트의 모든 파일에 타입 안전성 검사를 해줘

# 작업이 클라우드로 전송되고, 터미널은 자유!
# 진행 상황은 claude.ai에서 확인 가능
```

**언제 쓰나요?**
- 시간이 오래 걸리는 작업을 맡기고 다른 일 할 때
- 터미널을 닫아야 할 때 (퇴근, 이동 등)
- 여러 작업을 동시에 돌리고 싶을 때

### 2. `/teleport` (`/tp`) — 포탈로 불러오기

claude.ai 웹에서 대화하던 세션을 **로컬 터미널로 가져옵니다**.

```bash
# 터미널에서 이렇게 입력하면:
/teleport

# 또는 줄여서:
/tp

# 웹에서 진행 중이던 세션 목록이 표시되고,
# 선택하면 터미널에서 이어서 작업 가능!
```

**언제 쓰나요?**
- 웹에서 코드 설계를 논의하다가 실제 코딩으로 전환할 때
- 웹에서는 할 수 없는 로컬 파일 편집, git 사용이 필요할 때
- 클라우드에서 실행시킨 `&` 작업의 결과를 가져올 때

### 3. `/rc` (Remote Control) — 원격 리모컨

로컬에서 실행 중인 세션에 **다른 기기에서 원격 접속**합니다.

```bash
# 터미널에서 실행 중인 세션에 대해:
/rc

# 세션 ID가 표시되고, 다른 기기에서 접속 가능
# 세션 자체는 원래 터미널에 그대로 있음
```

**언제 쓰나요?**
- 데스크탑에서 작업 중인데 태블릿으로 확인하고 싶을 때
- 팀원에게 내 세션을 잠시 보여주고 싶을 때
- 이동 중에 진행 상황을 모바일로 체크할 때

---

## 비교 표

| 기능 | 방향 | 비유 | 세션 위치 변경 |
|------|------|------|--------------|
| `&` prefix | 터미널 → 클라우드 | 포탈로 보내기 | 클라우드로 이동 |
| `/teleport` | 클라우드 → 터미널 | 포탈로 불러오기 | 로컬로 이동 |
| `/rc` | 원격 → 로컬 (접속) | 리모컨 | 이동 없음 (원격 접속) |

---

## 전체 흐름 한눈에 보기

```
[터미널에서 작업 시작]
      |
      |--- 오래 걸리는 작업? ---> & prefix로 클라우드에 보내기
      |                              |
      |                              v
      |                         [claude.ai에서 진행 확인]
      |                              |
      |                              |--- 결과 가져오기? ---> /teleport
      |                              |
      |                              v
      |--- 웹에서 논의 중? -------> /teleport로 터미널에서 이어하기
      |
      |--- 다른 기기에서 보기? ---> /rc로 원격 접속
      |
      v
[어디서든 작업 계속!]
```
