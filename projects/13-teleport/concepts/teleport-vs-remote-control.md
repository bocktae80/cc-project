# 텔레포트 vs 리모트 컨트롤 — 이동과 조종의 차이

> 세션을 직접 옮기는 것과, 멀리서 조종하는 것은 다릅니다

---

## 비유: 이사 vs TV 리모컨

```
Teleport = 이사                         Remote Control = TV 리모컨
---------------------------             ---------------------------

  [서울에서 살다가]                        [거실에 TV가 있고]
       |                                      |
       | 짐을 싸서                              | 리모컨을 들고
       | 부산으로 이동!                           | 안방으로 이동
       |                                      |
       v                                      v
  [부산에서 생활]                           [안방에서 TV 조종]

  --> 나(세션)가 장소를 바꿈                --> TV(세션)는 그대로,
  --> 서울에는 더 이상 없음                     내가 다른 곳에서 조종
```

---

## 텔레포트 (`/teleport`)

세션 **자체**를 다른 환경으로 옮기는 것입니다.

```
  Before                              After
  ======                              =====

  +------------------+                +------------------+
  | claude.ai (웹)   |                | claude.ai (웹)   |
  |                  |                |                  |
  | [세션 A] 작업 중  |   /teleport   | (비어 있음)       |
  |                  | ----------->  |                  |
  +------------------+                +------------------+

                                      +------------------+
                                      | Terminal (로컬)   |
                                      |                  |
                                      | [세션 A] 여기로!  |
                                      |                  |
                                      +------------------+
```

**핵심 특징:**
- 세션이 웹에서 터미널로 **완전히 이동**
- 이동 후 웹에서는 해당 세션에 접근 불가
- 로컬 파일, git 등 **터미널 도구를 사용 가능**해짐
- 대화 이력이 모두 함께 이동

### 사용 시나리오

```
1. claude.ai에서 코드 설계를 논의
2. "이제 실제 코딩하자!" → /teleport
3. 터미널에서 파일 편집, git commit, 테스트 실행
```

---

## 리모트 컨트롤 (`/rc`)

세션은 **원래 위치에 그대로** 두고, 다른 곳에서 **원격 접속**하는 것입니다.

```
  Session Location                    Remote Access
  ================                    =============

  +------------------+                +------------------+
  | Terminal (로컬)   |                | Terminal (로컬)   |
  |                  |                |                  |
  | [세션 A] 실행 중  |     /rc       | [세션 A] 실행 중  |
  |                  | ----------->  |                  |
  +------------------+                +------------------+
                                             ^
                                             | 원격 접속
                                      +------------------+
                                      | 다른 기기 (웹)    |
                                      |                  |
                                      | [원격 뷰어]       |
                                      | 조종 가능!        |
                                      +------------------+
```

**핵심 특징:**
- 세션은 원래 터미널에 **그대로 존재**
- 다른 기기에서 **관찰 + 조종** 가능
- 여러 사람이 **동시에 접속** 가능
- 원격 접속을 끊어도 세션에 영향 없음

### 사용 시나리오

```
1. 데스크탑 터미널에서 리팩토링 작업 실행
2. 커피 사러 가면서 → /rc로 모바일 접속
3. 진행 상황 확인, 필요하면 추가 지시
4. 돌아와서 데스크탑에서 계속 작업
```

---

## 비교 표

| 비교 항목 | `/teleport` | `/rc` |
|-----------|------------|-------|
| 비유 | 이사 (세션이 이동) | TV 리모컨 (원격 조종) |
| 세션 위치 | **변경됨** (웹 → 터미널) | **변경 없음** (원래 위치 유지) |
| 원래 위치 | 세션 없어짐 | 세션 그대로 |
| 방향 | 웹 → 터미널 | 터미널 ← 원격 접속 |
| 목적 | 로컬 도구 사용 (파일, git) | 원격 모니터링 + 조종 |
| 동시 접속 | 불가 (하나의 위치에만) | 가능 (여러 기기에서) |
| 사용 상황 | 설계→구현 전환 | 이동 중 확인, 팀 공유 |

---

## 선택 가이드

```
질문: 세션을 다른 환경으로 완전히 옮기고 싶은가?
|
|-- 예 (웹에서 논의 → 터미널에서 구현)
|   --> /teleport 사용
|
|-- 아니요 (세션은 그대로, 다른 곳에서 보기만)
    |
    |-- 다른 기기에서 확인/조종하고 싶다
    |   --> /rc 사용
    |
    |-- 작업을 클라우드에서 실행하고 싶다
        --> & prefix 사용
```

## 세 가지 기능 조합 예시

```
09:00  [데스크탑 터미널]
       & 전체 코드 리팩토링 해줘        <- 클라우드로 보내기
       (터미널 자유)

10:00  [카페에서 모바일]
       claude.ai에서 진행 상황 확인     <- 웹에서 모니터링

12:00  [카페에서 노트북]
       /teleport                      <- 세션을 노트북으로 가져옴
       로컬에서 결과 확인 + 추가 작업

14:00  [노트북에서 계속 작업 중]
       /rc                            <- 리모컨 공유
       팀원에게 세션 보여주기

15:00  [데스크탑 복귀]
       /teleport 로 다시 가져오기       <- 데스크탑에서 마무리
```
