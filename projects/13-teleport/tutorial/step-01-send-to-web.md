# Step 1: & prefix로 클라우드에 보내기

> 시간이 오래 걸리는 작업을 클라우드로 보내고, 터미널은 자유롭게!

---

## & prefix란?

터미널에서 프롬프트 앞에 `&`를 붙이면, 그 작업이 **클라우드에서 실행**됩니다.
마치 게임에서 포탈에 아이템을 넣으면 다른 장소로 보내지는 것처럼요!

```
일반 실행                              & prefix 실행
----------                             ----------------

> 이 코드를 리팩토링해줘               > & 이 코드를 리팩토링해줘

[터미널에서 실행]                       [클라우드로 전송!]
[터미널 사용 불가...]                    [터미널 즉시 사용 가능!]
[기다려야 함...]                        [다른 작업 가능!]
[완료!]                                [claude.ai에서 결과 확인]
```

---

## 실습: 첫 번째 클라우드 전송

### 1. 터미널에서 & prefix 사용

```bash
# Claude Code가 실행된 상태에서:

> & 이 프로젝트의 README.md 파일들을 모두 읽고 개선점을 정리해줘
```

### 예상 출력

```
Sending task to cloud...

Task sent successfully!
Session ID: ses_abc123def456

View progress at: https://claude.ai/chat/ses_abc123def456
Or run /teleport to bring it back when done.

>
```

터미널에 `>` 프롬프트가 바로 돌아왔어요! 작업은 클라우드에서 계속 진행 중입니다.

### 2. 진행 상황 확인

claude.ai 웹사이트에 접속하면 해당 세션을 볼 수 있어요:

```
https://claude.ai/chat/ses_abc123def456
```

웹에서 보이는 화면:

```
+--------------------------------------------------+
|  Cloud Session: ses_abc123def456                  |
|  Status: Running...                               |
|--------------------------------------------------|
|                                                  |
|  사용자: 이 프로젝트의 README.md 파일들을 모두      |
|         읽고 개선점을 정리해줘                      |
|                                                  |
|  Claude: README.md 파일들을 분석하겠습니다...       |
|          (진행 중)                                 |
|                                                  |
+--------------------------------------------------+
```

### 3. 결과 가져오기

작업이 완료되면, 터미널에서 `/teleport`로 결과를 가져올 수 있어요:

```bash
> /teleport
# 또는
> /tp

# 클라우드 세션 목록이 표시됨
# 완료된 세션을 선택하면 터미널에서 결과 확인!
```

---

## & prefix의 규칙

### 사용법

```
& <프롬프트 내용>
```

맨 앞에 `&`와 공백 하나를 붙이면 됩니다. 그게 끝이에요!

### 할 수 있는 것

| 작업 | 예시 |
|------|------|
| 코드 분석 | `& 이 프로젝트의 아키텍처를 분석해줘` |
| 리팩토링 | `& src/ 폴더의 모든 파일에서 any 타입을 제거해줘` |
| 문서 작성 | `& API 문서를 작성해줘` |
| 코드 리뷰 | `& 최근 커밋 10개를 리뷰해줘` |
| 테스트 생성 | `& 모든 유틸 함수에 대한 테스트를 작성해줘` |

### 주의할 점

```diff
+ 시간이 오래 걸리는 작업에 적합
+ 터미널을 닫아도 작업 계속 진행
+ 여러 개를 동시에 보낼 수 있음
- 로컬 파일 수정은 클라우드에서 직접 못 함
- 실시간 대화가 필요한 경우에는 부적합
```

## 실전 예시: 대규모 리팩토링

프로젝트에 파일이 100개가 넘고, 전부 검사해야 한다면?

```bash
# 터미널에서:
> & 이 프로젝트의 모든 TypeScript 파일에서
    1. any 타입을 unknown으로 변경
    2. 누락된 반환 타입 추가
    3. 사용하지 않는 import 제거
    변경 사항을 파일별로 정리해줘
```

```
Sending task to cloud...
Task sent! Session ID: ses_xyz789
View at: https://claude.ai/chat/ses_xyz789
```

이제 터미널에서 다른 작업을 계속하면 돼요:

```bash
# 다른 작업을 계속:
> git log --oneline -5
> 오늘 할 일 목록을 정리해줘
```

30분 후, 클라우드 작업이 완료되면 결과를 가져옵니다:

```bash
> /tp
# ses_xyz789 선택
# 결과 확인 + 필요하면 추가 작업!
```
