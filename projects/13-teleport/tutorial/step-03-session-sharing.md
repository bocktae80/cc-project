# Step 3: 세션 공유 + /rc 리모트 컨트롤

> 세션을 팀원과 공유하고, 다른 기기에서 원격 조종하기

---

## /rc (Remote Control)란?

로컬 터미널에서 실행 중인 세션에 **다른 기기에서 원격으로 접속**하는 기능입니다.
TV 리모컨처럼, 세션(TV)은 원래 자리에 있고 리모컨(다른 기기)으로 조종하는 거예요.

```
세션이 있는 곳 (데스크탑)              원격 접속 (다른 기기)
---------------------------          ---------------------------

  +------------------+               +------------------+
  | Terminal          |               | 노트북/태블릿/폰  |
  |                  |               |                  |
  | [세션 실행 중]    | <-- /rc ---  | [원격 접속!]      |
  | 파일 편집 중...   |               | 진행 확인 가능    |
  |                  |               | 추가 지시 가능    |
  +------------------+               +------------------+
```

---

## 실습: 리모트 컨트롤 시작하기

### 1. 세션에서 /rc 활성화

```bash
# 데스크탑 터미널에서 Claude Code 실행 중:
> /rc
```

### 예상 출력

```
Remote Control enabled!

Session ID: ses_abc123def456
Share URL:  https://claude.ai/rc/ses_abc123def456

Anyone with this URL can:
- View the session in real-time
- Send messages to the session

Share this URL with teammates to collaborate.
Press Ctrl+C to disable remote control.
```

### 2. 다른 기기에서 접속

공유 URL을 브라우저에서 열면:

```
+--------------------------------------------------+
|  Remote Control: ses_abc123def456                 |
|  Connected to: kent's terminal session            |
|--------------------------------------------------|
|                                                  |
|  [실시간 세션 내용이 표시됨]                        |
|                                                  |
|  사용자: src/utils.ts를 리팩토링해줘                |
|  Claude: 네, 리팩토링을 시작합니다...               |
|          [파일 수정 중...]                         |
|                                                  |
|  +--------------------------------------------+  |
|  | 메시지 입력...                    [Send]     |  |
|  +--------------------------------------------+  |
+--------------------------------------------------+
```

### 3. 원격에서 추가 지시

다른 기기에서 메시지를 보내면, 터미널 세션에 반영됩니다:

```
# 태블릿 브라우저에서 입력:
"utils.ts에서 날짜 관련 함수만 따로 date-utils.ts로 분리해줘"

# 데스크탑 터미널에 표시:
[Remote] 태블릿에서 메시지: "utils.ts에서 날짜 관련 함수만
         따로 date-utils.ts로 분리해줘"

Claude: 네, 날짜 관련 함수를 분리하겠습니다...
```

## 세션 공유하기

### 세션 ID 전달

팀원에게 세션을 공유하는 가장 간단한 방법은 **세션 ID 또는 URL을 전달**하는 것입니다.

```bash
# 현재 세션 정보 확인:
> /rc

# 표시된 URL을 팀원에게 전달:
# https://claude.ai/rc/ses_abc123def456
```

### 공유 시나리오

```
[나]                                 [팀원]
 |                                      |
 |  /rc로 세션 공유 URL 생성              |
 |                                      |
 |  "이 URL로 접속해봐"                   |
 |  ------ URL 전달 ------>             |
 |                                      | 브라우저에서 접속
 |                                      |
 |  [세션에서 작업 중]                     | [실시간으로 관찰]
 |                                      | [추가 지시 가능]
 |                                      |
 |  "이 부분 어떻게 생각해?"               | "거기에 에러 처리
 |  <------ 실시간 협업 ------>          |  추가하는 게 좋겠어"
```

## 보안 주의사항

세션을 공유할 때 꼭 주의할 점이 있어요:

```
+--------------------------------------------------+
|  주의! 세션에 이런 내용이 있으면 공유하지 마세요      |
|                                                  |
|  - API 키, 토큰, 비밀번호                          |
|  - .env 파일 내용                                 |
|  - 개인정보 (이름, 이메일, 전화번호)                  |
|  - 회사 내부 기밀 코드                              |
+--------------------------------------------------+
```

### 안전한 공유 체크리스트

| 항목 | 확인 |
|------|------|
| 세션에 API 키가 노출되지 않았는가? | [ ] |
| .env 파일 내용을 읽지 않았는가? | [ ] |
| 개인정보가 포함되지 않았는가? | [ ] |
| 신뢰할 수 있는 사람에게만 공유하는가? | [ ] |

## /rc 활용 팁

### 페어 프로그래밍

```
[개발자 A: 데스크탑]                 [개발자 B: 노트북]
  |                                    |
  | Claude Code 세션 실행               |
  | /rc로 공유                          | URL로 접속
  |                                    |
  | "이 함수를 구현해줘"                  | "테스트도 같이 작성해달라고 해봐"
  |                                    |
  |  ------ 함께 Claude 지시 ------>    |
  |                                    |
  | [코드 결과 함께 확인]                 | [실시간으로 같이 봄]
```

### 이동 중 모니터링

```
[데스크탑: 장시간 작업 실행 중]

  12:00  작업 시작: "전체 코드를 분석해줘"
  12:05  /rc 활성화

[모바일: 점심 먹으러 가면서]

  12:10  URL 접속 → 진행 상황 확인
  12:30  "완료됐네! 결과를 REPORT.md로 저장해줘"
  12:35  저장 확인

[데스크탑: 돌아와서]

  13:00  REPORT.md 열어서 결과 확인
```

## 텔레포트 vs 리모트 컨트롤 — 언제 뭘 쓸까?

| 상황 | 추천 기능 | 이유 |
|------|----------|------|
| 웹에서 설계 → 터미널에서 코딩 | `/teleport` | 세션을 완전히 옮겨야 함 |
| 데스크탑 세션을 모바일로 확인 | `/rc` | 세션은 데스크탑에 두고 원격 확인 |
| 팀원에게 세션 보여주기 | `/rc` | 여러 사람이 동시 접속 가능 |
| 클라우드에서 결과 가져오기 | `/teleport` | & prefix로 보낸 작업 가져오기 |
| 장시간 작업 중 외출 | `/rc` | 세션은 계속 실행, 모바일로 확인 |

## 전체 학습 정리

축하해요! 텔레포트의 세 가지 핵심 기능을 모두 배웠어요:

```
+--------------------------------------------------+

|  3. /rc         : 세션에 원격으로 접속하기          |

|                                                  |
+--------------------------------------------------+
```
