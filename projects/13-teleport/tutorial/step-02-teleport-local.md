# Step 2: /teleport로 로컬에 가져오기

> 웹에서 진행 중인 세션을 터미널로 순간이동!

---

## /teleport란?

claude.ai 웹에서 대화하던 세션을 **로컬 터미널로 가져오는** 명령어입니다.
줄여서 `/tp`로도 사용할 수 있어요.

```
Before                                 After
------                                 -----

  +------------------+                 +------------------+
  | claude.ai (웹)   |                 | claude.ai (웹)   |
  |                  |                 |                  |
  | [대화 진행 중]    |   /teleport    | (세션 이동 완료)  |
  | "코드 설계를      | ----------->  |                  |
  |  논의하고 있었음"  |                 +------------------+
  +------------------+
                                       +------------------+
                                       | Terminal (로컬)   |
                                       |                  |
                                       | [대화 이어서!]    |
                                       | 파일 편집 가능    |
                                       | git 사용 가능     |
                                       +------------------+
```

## 실습: 웹 세션을 터미널로 가져오기

### 시나리오

claude.ai에서 새로운 API 설계를 논의하고 있었어요. 이제 실제 코드를 작성할 차례입니다.

### 1. 텔레포트 실행

```bash
# Claude Code 터미널에서:
> /teleport
```

또는 줄여서:

```bash
> /tp
```

```
  3. [ses_ghi789] "버그 분석" (2시간 전, completed)

Select a session (1-3): _
```

### 3. 터미널에서 이어서 작업

이제 로컬 환경의 모든 기능을 사용할 수 있어요!

```bash

```

## 왜 /teleport가 필요한가?

### 웹(claude.ai)에서 할 수 있는 것 vs 할 수 없는 것

| 기능 | 웹 (claude.ai) | 터미널 (Claude Code) |
|------|:--------------:|:-------------------:|
| 대화, 질문 | O | O |
| 코드 설계 논의 | O | O |
| 로컬 파일 읽기/쓰기 | X | O |
| git commit/push | X | O |
| 테스트 실행 | X | O |
| npm/pip 명령어 | X | O |
| 로컬 MCP 서버 | X | O |

웹은 **대화와 설계**에 좋고, 터미널은 **실행과 구현**에 좋아요.
`/teleport`는 이 두 세계를 **연결하는 다리** 역할을 합니다!

## 실전 워크플로우

### 전형적인 텔레포트 패턴

```
[1단계: 웹에서 설계]
  claude.ai에서 자유롭게 논의
  - "이 기능은 어떻게 구현하면 좋을까?"
  - "이 아키텍처의 장단점은?"
  - "테스트 전략은?"

        |
        | /teleport
        v

[2단계: 터미널에서 구현]
  논의한 내용을 바로 코드로
  - 파일 생성/수정
  - 테스트 작성 + 실행
  - git commit + push

        |
        | (필요하면) & prefix로 다시 클라우드에
        v

[3단계: 웹에서 결과 확인]
  팀원과 결과 공유
  - 코드 리뷰 요청
  - 추가 설계 논의
```

### 구체적인 예시

```
09:00 [claude.ai 웹]
      "새로운 사용자 인증 시스템을 설계하자"
      → JWT vs 세션 비교, 아키텍처 논의

09:30 [터미널]
      /teleport
      → 세션 가져오기
      → "방금 논의한 JWT 기반으로 auth 모듈을 만들어줘"
      → 파일 생성, 테스트 작성

10:30 [터미널]
      & 전체 테스트 스위트를 실행하고 결과를 정리해줘
      → 클라우드로 보내기
      → 다른 작업 진행

11:00 [claude.ai 웹]
      테스트 결과 확인
      → 실패한 테스트 분석 논의
```

## /teleport의 옵션들

### 기본 사용

```bash
# 세션 목록 표시 후 선택
> /teleport

# 줄임말
> /tp
```

### 세션 ID로 직접 지정

```bash
# 특정 세션을 바로 가져오기
> /teleport ses_abc123
```

## 주의사항

```diff
+ 대화 이력이 모두 함께 이동
+ 웹에서 논의한 맥락을 터미널에서 그대로 활용
+ 로컬 파일, git 등 터미널 도구 사용 가능
- 텔레포트 후 웹에서 해당 세션 접근 불가
- 인터넷 연결 필요
```
