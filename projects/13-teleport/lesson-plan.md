# 튜터 지시서: 텔레포트

> 이 파일은 `/learn 13` 실행 시 튜터(클로드)가 읽는 지시서입니다. 사용자에게 직접 보여주지 마세요.

## 학습 목표

텔레포트, 리모트 컨트롤로 세션을 자유롭게 이동하는 방법을 이해한다.

## 핵심 비유

"게임에서 순간이동 포탈 써본 적 있죠? 클로드 코드도 비슷해요! 터미널에서 작업하다가 포탈을 열면 웹브라우저에서 이어하고, 다시 포탈로 돌아올 수 있어요. 심지어 친구에게 리모컨을 건네주면, 친구가 내 세션을 원격으로 조종할 수도 있어요!"

## 교육 흐름

### Step 1: & prefix로 클라우드에 보내기 (포탈 열기)

1. "오늘은 클로드 코드의 순간이동 기능을 배워볼 거예요!"
2. concepts/what-is-teleport.md를 Read로 읽어 참고
3. `&` prefix의 핵심:
   - 프롬프트 앞에 `&`를 붙이면 클라우드에서 실행
   - 터미널을 닫아도 계속 진행됨
   - 진행 상황은 claude.ai에서 확인 가능
4. "터미널에서 '포탈을 열어서 작업을 웹으로 보내는 거예요!' 시간이 오래 걸리는 작업을 보내놓고, 다른 일을 할 수 있어요."
5. tutorial/step-01을 Read로 읽어 실습 안내

**확인 질문**: "& prefix는 어떤 상황에서 유용할까요?"
→ 기대 답변: 오래 걸리는 작업을 보내놓고 다른 일을 할 때 / 터미널을 닫아야 할 때

### Step 2: /teleport로 세션 가져오기 (포탈로 불러오기)

1. "이번에는 반대 방향이에요. 웹에서 하던 작업을 터미널로 가져오는 거예요!"
2. `/teleport` (줄여서 `/tp`) 명령어 설명:
   - claude.ai에서 대화하던 세션을 로컬 터미널로 이동
   - 로컬 파일 시스템, git 등에 접근 가능해짐
   - 웹에서는 할 수 없는 직접적인 파일 편집이 가능
3. "웹에서 코드 설계를 논의하다가, 실제로 코딩할 때 포탈을 열어서 터미널로 가져오는 거예요!"
4. tutorial/step-02를 Read로 읽어 실습 안내

**확인 질문**: "/teleport는 왜 필요한가요? 그냥 웹에서 하면 안 되나요?"
→ 기대 답변: 웹에서는 로컬 파일을 직접 수정하거나 git을 사용할 수 없음

### Step 3: /rc 리모트 컨트롤 (원격 리모컨)

1. "마지막으로 리모트 컨트롤! 텔레포트가 '세션을 옮기는 것'이라면, 리모트 컨트롤은 '세션은 그대로 두고 원격으로 조종하는 것'이에요."
2. concepts/teleport-vs-remote-control.md를 Read로 읽어 비교
3. `/rc` 명령어의 핵심:
   - 로컬에서 실행 중인 세션에 원격으로 접속
   - 세션 자체는 이동하지 않음
   - 여러 기기에서 같은 세션을 관찰/조종 가능
4. "TV 리모컨처럼 생각하면 돼요. TV(세션)는 거실에 있고, 리모컨(rc)으로 다른 방에서 채널을 돌리는 거예요!"
5. tutorial/step-03을 Read로 읽어 실습 안내

### Step 4: 세션 공유 + 마무리 퀴즈

1. 세션 공유 개념:
   - 세션 ID를 팀원에게 전달하면 같은 세션을 볼 수 있음
   - 보안 주의: 민감한 정보가 있는 세션은 공유 주의
2. 실전 시나리오 정리:
   - 장시간 리팩토링 → `&` prefix로 보내기
   - 웹에서 설계 논의 → `/teleport`로 구현 전환
   - 이동 중 모바일로 확인 → `/rc`로 접속
3. examples/ 폴더의 3가지 예제 소개

퀴즈:
1. "& prefix를 붙이면 어떤 일이 일어나나요?"
   → 정답: 작업이 클라우드에서 실행됨 (터미널을 닫아도 계속 진행)
2. "/teleport와 /rc의 차이는?"
   → 정답: /teleport는 세션 자체를 옮기고, /rc는 세션을 그대로 두고 원격 접속함
3. "웹에서 코드 설계를 논의하다가 실제 코딩으로 전환할 때 어떤 기능을 쓰나요?"
   → 정답: /teleport (웹 세션을 로컬 터미널로 가져옴)
4. "세션을 공유할 때 주의할 점은?"
   → 정답: 민감한 정보(API 키, 비밀번호 등)가 포함된 세션은 공유 주의

### 마무리

"텔레포트를 배웠어요! 이제 세션을 장소에 구애받지 않고 자유롭게 이동할 수 있어요.
정리하면:
- `&` prefix: 작업을 클라우드로 보내기 (포탈로 보내기)
- `/teleport` (`/tp`): 웹 세션을 터미널로 가져오기 (포탈로 불러오기)
- `/rc`: 세션을 원격으로 조종하기 (리모컨)

이 세 가지를 조합하면, 어디서든 어떤 기기에서든 작업을 이어갈 수 있어요!"
