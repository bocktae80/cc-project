# 예제: 리모트 컨트롤로 원격 조종

> 데스크탑에서 실행 중인 세션을 모바일/태블릿에서 확인하고 조종하기

---

## 실습 목표

```
[데스크탑: 세션 실행 중]
        |
        | /rc 활성화
        v
[모바일/태블릿에서 접속]
        |
        | 진행 확인 + 추가 지시
        v
[데스크탑에서 계속 작업]
```

---

## 시나리오

데스크탑에서 대규모 코드 분석 작업을 시작했습니다. 점심 먹으러 가면서 모바일로 진행 상황을 확인하고, 필요하면 추가 지시를 보내고 싶어요.

---

## Phase 1: 데스크탑에서 작업 시작 (2분)

### 장시간 작업 실행

```bash
# Claude Code 터미널에서:
> 이 프로젝트의 모든 파일을 분석하고:
  1. 코드 품질 점수 (1-10)
  2. 개선이 필요한 부분
  3. 리팩토링 우선순위
  를 파일별로 정리해줘
```

Claude가 분석을 시작합니다... 시간이 좀 걸릴 거예요.

### /rc 활성화

```bash
# 분석이 진행 중인 상태에서:
> /rc
```

```
Remote Control enabled!

Session ID: ses_analysis_001
Share URL:  https://claude.ai/rc/ses_analysis_001

Anyone with this URL can view and interact with this session.
```

URL을 메모하거나 본인에게 메시지로 보냅니다.

---

## Phase 2: 모바일에서 원격 접속 (이동 중)

### 브라우저에서 접속

모바일 브라우저에서 공유 URL을 엽니다:

```
https://claude.ai/rc/ses_analysis_001
```

### 화면에 표시되는 내용

```
+------------------------------------------+
|  Remote Session: ses_analysis_001         |
|  Status: Running                         |
|------------------------------------------|
|                                          |
|  Claude: 파일 분석 진행 중... (35/50)     |
|                                          |
|  현재까지 결과:                            |
|  src/utils/date.ts     - 8/10            |
|  src/utils/string.ts   - 6/10            |
|  src/api/client.ts     - 5/10 (개선 필요) |
|  src/api/auth.ts       - 7/10            |
|  ...                                     |
|                                          |
|  +------------------------------------+  |
|  | 메시지 입력...             [Send]   |  |
|  +------------------------------------+  |
+------------------------------------------+
```

### 추가 지시 보내기

진행 상황을 보다가, 추가로 요청할 게 있으면 메시지를 보냅니다:

```
[모바일에서 입력]
"보안 관련 파일은 특히 자세히 봐줘.
 특히 auth.ts와 token.ts의 보안 취약점을 집중 분석해줘"
```

### 데스크탑 터미널에 표시

```
[Remote message received]
"보안 관련 파일은 특히 자세히 봐줘..."

Claude: 네, 보안 관련 파일을 추가 분석하겠습니다.
        auth.ts와 token.ts에 대해 보안 집중 리뷰를 진행합니다...
```

## Phase 3: 팀원과 함께 보기 (선택)

### 팀원에게 URL 공유

```
나: "지금 코드 분석 결과 같이 보자"
팀원: "URL 보내줘"
나: https://claude.ai/rc/ses_analysis_001
```

### 여러 사람이 동시 접속

```
+------------------+
| 데스크탑 (나)     | --- 세션 실행 중
+------------------+
        ^
        |  /rc
        |
+------------------+     +------------------+
| 모바일 (나)       |     | 노트북 (팀원)     |
| 진행 상황 확인     |     | 실시간 관찰       |
+------------------+     +------------------+
```

팀원도 메시지를 보낼 수 있어요:

```
[팀원이 노트북에서 입력]
"client.ts에서 에러 처리가 빠진 것 같은데, 그것도 분석해줘"

[데스크탑 터미널에 표시]
[Remote message from 팀원]
"client.ts에서 에러 처리가 빠진 것 같은데..."

Claude: 좋은 지적이네요. client.ts의 에러 처리를 분석하겠습니다...
```

## Phase 4: 데스크탑 복귀 + 마무리 (2분)

점심을 먹고 돌아와서 데스크탑에서 결과를 확인합니다:

```bash
# 이미 데스크탑 터미널에 결과가 있음:

Claude: 분석이 완료되었습니다!

=== 코드 품질 리포트 ===

전체 점수: 7.2 / 10

Top 3 개선 필요 파일:
1. src/api/client.ts (5/10) - 에러 처리 누락, any 타입 다수
2. src/utils/string.ts (6/10) - 중복 함수, 타입 안전성
3. src/components/Form.tsx (6/10) - 접근성 누락

보안 집중 리뷰 (모바일에서 요청):
- auth.ts: 토큰 만료 검증 로직 개선 필요
- token.ts: 하드코딩된 시크릿 발견!

리팩토링 우선순위:
1순위: client.ts (보안 + 품질)
2순위: auth.ts (보안)
3순위: string.ts (코드 품질)
```

## /rc 활용 팁

### 효과적인 리모트 컨트롤 사용법

| 팁 | 설명 |
|-----|------|
| URL 저장 | 자주 쓰는 세션 URL을 북마크해두기 |
| 짧은 메시지 | 모바일에서는 간결하게 지시하기 |
| 관찰 위주 | 복잡한 작업은 데스크탑 복귀 후 진행 |
| 보안 주의 | URL을 신뢰할 수 있는 사람에게만 공유 |

### /rc가 특히 유용한 상황

```
1. 장시간 작업 + 이동
   - 빌드, 테스트, 분석 등 오래 걸리는 작업 모니터링

2. 페어 프로그래밍
   - 팀원과 같은 세션에서 실시간 협업

3. 코드 리뷰
   - 작업 결과를 팀원에게 실시간으로 보여주기

4. 발표/데모
   - Claude Code 작업 과정을 여러 사람에게 라이브로 보여주기
```

## 학습 포인트

이 예제를 통해 배운 것:

1. **`/rc` 활성화**: 한 줄 명령으로 원격 접속 활성화
2. **원격 모니터링**: 다른 기기에서 실시간 진행 상황 확인
3. **원격 지시**: 모바일에서도 추가 지시를 보낼 수 있음
4. **팀 협업**: URL만 공유하면 여러 사람이 동시 접속
5. **보안**: 민감한 내용이 있는 세션은 URL 공유 주의

> 핵심 교훈: 세션은 한 곳에, 접속은 어디서든!
> 리모트 컨트롤로 장소에 구애받지 않고 작업을 관리하세요.
