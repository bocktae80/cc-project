# 튜터 지시서: 백그라운드 에이전트

> 이 파일은 `/learn 16` 실행 시 튜터(클로드)가 읽는 지시서입니다. 사용자에게 직접 보여주지 마세요.

## 학습 목표

포그라운드와 백그라운드의 차이를 이해하고, run_in_background과 TaskOutput을 사용해 여러 작업을 동시에 처리하는 방법을 배운다.

## 핵심 비유

"요리사가 국 끓이면서 동시에 야채도 썰듯, 여러 작업을 동시에! 국이 끓는 동안 가만히 기다리지 않고, 다른 요리를 준비하는 거예요."

## 연습 파일

- 없음 (실습 위주)

## 교육 흐름

### Step 1: 포그라운드 vs 백그라운드

1. "컴퓨터에서 작업할 때, 하나씩만 하면 느리잖아요?"
2. 두 가지 실행 방식 설명:
   - **포그라운드**: 앞에서 실행, 끝날 때까지 기다려야 함
   - **백그라운드**: 뒤에서 실행, 기다리는 동안 다른 일 가능
3. 비유 활용:
   - 포그라운드 = 라면 끓이면서 앞에서 지켜보기
   - 백그라운드 = 밥솥에 밥 넣고 다른 반찬 만들기
4. "밥솥이 알아서 밥을 하는 동안, 여러분은 반찬을 만들 수 있어요!"

### Step 2: run_in_background 파라미터와 TaskOutput 사용법

1. Bash 도구의 run_in_background 파라미터:
   - `run_in_background: true`로 설정하면 명령이 백그라운드에서 실행
   - 결과는 나중에 자동으로 알림이 옴
   - TaskOutput으로 중간 결과 확인 가능
2. 실습 예시:
   - `npm test`를 백그라운드로 실행하면서 코드 수정
   - 개발 서버를 백그라운드로 돌리면서 코드 작성
3. "밥솥 타이머가 울리면 확인하듯, 작업이 끝나면 알림이 와요!"

### Step 3: Ctrl+C 처리와 작업 관리

1. 백그라운드 작업 관리:
   - 실행 중인 작업 확인
   - 필요 없는 작업 중단 (Ctrl+C / TaskStop)
   - 여러 작업 동시 관리 패턴
2. 주의사항:
   - 백그라운드 작업이 너무 많으면 시스템이 느려질 수 있음
   - 장시간 작업은 & prefix(텔레포트, 13번)와 비교
3. "요리사도 한꺼번에 10가지 요리를 하면 정신없겠죠? 2-3개가 적당해요!"

### Step 4: 마무리 퀴즈

퀴즈:
1. "포그라운드와 백그라운드의 가장 큰 차이는?"
   → 정답: 포그라운드는 끝날 때까지 기다려야 하고, 백그라운드는 기다리는 동안 다른 작업 가능
2. "백그라운드 작업의 결과를 확인하는 방법은?"
   → 정답: 작업 완료 시 자동 알림이 오거나, TaskOutput으로 중간 확인 가능
3. "& prefix(텔레포트)와 run_in_background의 차이는?"
   → 정답: &는 클라우드에서 실행, run_in_background는 로컬에서 실행
