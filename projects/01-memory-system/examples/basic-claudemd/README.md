# 예제 1: CLAUDE.md 만들기 ⭐

> CLAUDE.md가 있을 때와 없을 때, 클로드의 답변이 어떻게 달라지는지 확인해보자!

## 이 예제에서 배우는 것

- CLAUDE.md가 뭔지
- CLAUDE.md가 있으면 뭐가 달라지는지
- 나만의 CLAUDE.md 만드는 법

---

## 실험: 전 vs 후

### BEFORE — CLAUDE.md 없이

`before/` 폴더를 보세요. `app.js` 파일만 있고 CLAUDE.md는 없습니다.

이 상태에서 클로드에게 "함수 하나 추가해줘"라고 하면:

```
나: "인사하는 함수 추가해줘"

클로드: (영어로 답할 수도, 한국어로 답할 수도 있음)
       (코드 스타일도 매번 다를 수 있음)
```

클로드는 **아무 맥락 없이** 답하기 때문에, 매번 스타일이 다를 수 있습니다.

### AFTER — CLAUDE.md 추가 후

`after/` 폴더를 보세요. `CLAUDE.md`와 `app.js`가 있습니다.

CLAUDE.md에 이렇게 적혀있으면:

```markdown
# 프로젝트 규칙

- 코드 주석은 한국어로 작성
- 함수명은 camelCase 사용
- 모든 함수에 JSDoc 주석 추가
```

같은 질문을 해도:

```
나: "인사하는 함수 추가해줘"

클로드: (한국어 주석으로, camelCase로, JSDoc과 함께 작성!)
```

**CLAUDE.md에 적힌 규칙을 자동으로 따릅니다!**

---

## 따라하기

### 1단계: before 폴더에서 테스트

```bash
# before 폴더로 이동
cd before/

# 클로드 시작
claude

# 질문하기
> "greet 함수를 추가해줘. 이름을 받아서 인사말을 반환하는 함수야"
```

클로드가 어떻게 답하는지 관찰하세요.
- 주석이 영어인가, 한국어인가?
- 함수명 스타일은?
- JSDoc이 있나?

### 2단계: after 폴더에서 테스트

```bash
# after 폴더로 이동
cd ../after/

# 클로드 시작 (CLAUDE.md가 자동으로 로드됨!)
claude

# 똑같은 질문하기
> "greet 함수를 추가해줘. 이름을 받아서 인사말을 반환하는 함수야"
```

이번에는 CLAUDE.md의 규칙을 따라 답변이 달라질 거예요!

### 3단계: 직접 만들어보기

`exercise/` 폴더에서 나만의 CLAUDE.md를 만들어보세요.

```bash
cd ../exercise/
```

`CLAUDE.md` 파일을 만들고, 원하는 규칙을 적어보세요:

```markdown
# 내 프로젝트 규칙

- (여기에 원하는 규칙을 적어보세요!)
```

---

## 예시: 이런 규칙을 적어볼 수 있어요

```markdown
# 내 프로젝트

## 코드 스타일
- 변수명은 한글도 OK (교육용 프로젝트)
- console.log 대신 한국어 메시지 출력

## 답변 방식
- 설명은 항상 한국어로
- 코드에 주석을 꼭 달아줘
- 어려운 용어는 쉽게 풀어서 설명
```

---

## 핵심 정리

```
CLAUDE.md = 클로드에게 주는 "프로젝트 설명서"

✅ 있으면: 규칙을 따라 일관된 답변
❌ 없으면: 매번 다른 스타일

만드는 법: 프로젝트 루트에 CLAUDE.md 파일 생성
또는: 클로드에서 /init 명령어 사용
```
