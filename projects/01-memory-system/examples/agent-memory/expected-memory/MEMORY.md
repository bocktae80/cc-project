# 코드 리뷰 에이전트 — 학습 메모리 (예시)

> 이 파일은 에이전트가 실제로 쌓을 메모리의 "예상" 모습입니다.
> 실제 에이전트를 사용하면 이와 비슷한 MEMORY.md가 자동 생성됩니다.

## 프로젝트에서 발견한 패턴

### 좋은 패턴
- JSDoc 주석을 사용하는 함수가 있음
- 입력값 유효성 검사를 하는 함수 패턴 (`isValidName`)
- 화살표 함수와 const 사용

### 자주 보이는 문제
- `var` 대신 `const`/`let`을 사용해야 하는 경우가 있음
- 함수명이 불명확한 경우가 있음 (예: `doStuff`)
- 매직 넘버 사용 (상수로 추출 필요)
- 에러 처리가 누락된 함수가 있음
- 콜백 패턴 → async/await으로 개선 가능

## 리뷰 시 중점 확인 사항
- 변수/함수 네이밍이 명확한지
- const/let을 적절히 사용하는지
- 에러 처리가 되어 있는지
- 중복 코드가 있는지
