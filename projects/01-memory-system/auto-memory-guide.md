# 자동 메모리 관찰 가이드 ⭐⭐

> 클로드가 알아서 기억하는 "자동 메모리"를 관찰해보자!

## 자동 메모리란?

지금까지 본 메모리(CLAUDE.md 등)는 **내가 직접 작성**하는 것이었습니다.
자동 메모리는 다릅니다 — **클로드가 알아서** 대화 내용을 정리해서 저장합니다.

### 비유로 이해하기

| 수동 메모리 | 자동 메모리 |
|------------|------------|
| 내가 노트에 적는 것 | 녹음기가 자동으로 요약해주는 것 |
| CLAUDE.md에 규칙을 적음 | 클로드가 대화를 요약해서 저장 |
| 내용을 내가 정함 | 클로드가 중요한 것을 골라서 기록 |

---

## 어떻게 동작하나?

### 저장 과정

```
대화 시작
    │
    ▼
충분한 대화 진행 (약 10,000 토큰)
    │
    ▼
클로드가 자동으로 핵심 내용 추출
    │
    ▼
요약을 파일로 저장
    │
    ▼
이후 약 5,000 토큰마다 또는
도구 사용 3회마다 업데이트
```

### 불러오기 과정

```
다음 대화 시작
    │
    ▼
같은 프로젝트 폴더에서 시작하면
    │
    ▼
이전 세션 요약을 자동으로 불러옴
    │
    ▼
"Recalled N memories" 메시지 표시
```

## 직접 관찰해보기

### 1단계: 터미널 메시지 확인

클로드 코드를 시작할 때 터미널을 주의 깊게 보세요:

```
$ claude

  Recalled 3 memories                    ← 이전 메모리를 불러왔다는 뜻!
```

대화를 끝낼 때:

```
  Wrote 2 memories                       ← 이번 대화를 저장했다는 뜻!
```

### 2단계: 저장 위치 확인

자동 메모리는 이 경로에 저장됩니다:

```
~/.claude/projects/<프로젝트-경로-해시>/
```

터미널에서 확인해볼 수 있습니다:

```bash
# 자동 메모리 폴더 목록 보기
ls ~/.claude/projects/
```

> **참고:** 폴더 이름이 긴 해시값으로 되어 있어서 읽기 어렵습니다.
> 이건 정상이에요 — 프로젝트 경로를 해시로 변환한 것입니다.

### 3단계: 자동 메모리 내용 읽어보기

저장된 메모리 파일을 열어보면 이런 내용이 있습니다:

```markdown
# 세션 요약

## 프로젝트 컨텍스트
- TypeScript 프로젝트, ESM 사용
- 테스트 프레임워크: Vitest

## 이번 세션에서 한 일
- 로그인 기능 버그 수정
- 에러 핸들링 추가

## 중요 결정
- JWT 대신 세션 쿠키 사용하기로 함
```

## 자동 메모리 vs 수동 메모리

| 비교 | 자동 메모리 | 수동 메모리 (CLAUDE.md) |
|------|-----------|----------------------|
| **누가 쓰나** | 클로드가 자동으로 | 내가 직접 |
| **무엇을 기록** | 대화 내용 요약 | 규칙, 컨벤션, 컨텍스트 |
| **정확도** | 클로드가 판단 (놓칠 수 있음) | 내가 쓴 그대로 |
| **용도** | "지난번에 뭐 했더라?" | "이 프로젝트에서는 이렇게 해" |
| **추천 상황** | 긴 작업의 연속성 유지 | 항상 지켜야 할 규칙 |

### 핵심 포인트

> 자동 메모리는 **편리하지만**, 중요한 규칙은 반드시 **CLAUDE.md에 직접 작성**하세요.
> 자동 메모리는 "있으면 좋은 것", CLAUDE.md는 "반드시 있어야 하는 것"입니다.

---

## 제한 사항

- **구독 필요:** Pro 또는 Max 구독에서 사용 가능합니다
- **자동 판단:** 클로드가 중요하다고 판단한 것만 저장하므로, 모든 내용이 기록되지는 않습니다
- **직접 수정 불가:** 자동 메모리 내용을 직접 편집하는 것은 권장되지 않습니다

```
- "Recalled/Wrote N memories" 메시지로 확인
- 편리하지만 중요한 건 CLAUDE.md에!
```
