# 첫 번째 훅 만들기 ⭐

> 가장 간단한 훅으로 시작해봐요! 📝

---

## 이번에 만들 것

클로드가 **Write 도구**를 사용할 때마다 "📝 파일이 작성됩니다!" 메시지가 자동으로 표시되는 훅을 만듭니다.

```
💡 비유: 편의점 문 열 때 "띵동~" 소리

   손님이 문을 열면 → 🔔 "띵동~" 자동 재생
   클로드가 Write하면 → 📝 "파일이 작성됩니다!" 자동 표시
```

---

## 따라하기

### 1단계: settings.json 확인하기

이 폴더의 `.claude/settings.json` 파일을 열어보세요:

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write",
        "hook": "echo '📝 파일이 작성됩니다!'"
      }
    ]
  }
}
```

각 부분의 의미:

| 요소 | 값 | 의미 |
|------|-----|------|
| `hooks` | `{}` | 훅 설정의 시작 |
| `PreToolUse` | `[]` | 도구 사용 **전**에 실행할 훅 목록 |
| `matcher` | `"Write"` | **Write** 도구에만 반응 |
| `hook` | `"echo '...'"` | 실행할 명령어 |

### 2단계: 이 폴더에서 클로드 코드 실행

```bash
# 이 폴더로 이동
cd projects/06-hooks/examples/first-hook/

# 클로드 코드 실행
claude
```

### 3단계: 파일 작성 요청하기

클로드에게 이렇게 말해보세요:

```
test.txt 파일에 "안녕하세요!"라고 써줘
```

### 4단계: 결과 확인

클로드가 Write 도구를 사용하기 전에 다음 메시지가 표시됩니다:

```
📝 파일이 작성됩니다!
```

그리고 나서 파일이 정상적으로 작성됩니다. 훅이 `exit 0`(기본값)을 반환하므로 도구 사용이 **허용**됩니다.

---

## 어떻게 동작하나요?

```
1. 사용자: "test.txt에 써줘"
2. 클로드: Write 도구 사용 결정
3. 🪝 PreToolUse 훅 실행!
   → matcher가 "Write"이므로 이 훅이 실행됨
   → echo '📝 파일이 작성됩니다!' 실행
   → exit 0 (기본값) → 허용!
4. Write 도구 실행 → 파일 작성 완료
5. 클로드: "파일을 작성했습니다"
```
