# 문제 해결 가이드 🔧

> 스킬이 안 돼요! 걱정 마세요, 대부분 간단한 실수예요 😊

---

## 🚨 자주 겪는 문제 5가지

### 1. 스킬이 목록에 안 나와요!

**증상**: `/help`를 쳤는데 내가 만든 스킬이 목록에 없음

**원인과 해결**:

| 원인 | 확인 방법 | 해결 |
|------|----------|------|
| 폴더 위치가 틀림 | 폴더 경로 확인 | `.claude/commands/`에 있어야 함 |
| 폴더 이름 오타 | `ls -la .claude/` | `commands`인지 확인 (복수형!) |
| 점(.) 빠짐 | `ls -la` | `.claude`여야 함 (`claude` ❌) |
| 파일 확장자 틀림 | 파일명 확인 | `.md`여야 함 (`.txt` ❌) |
| 다른 폴더에서 실행 | `pwd` 확인 | 프로젝트 루트에서 실행 |

**올바른 구조**:
```
내-프로젝트/          ← 여기서 claude 실행!
├── .claude/          ← 점(.) 필수!
│   └── commands/     ← 복수형(s) 필수!
│       └── review.md ← .md 확장자 필수!
└── src/
```

**자가 진단 명령어**:
```bash
# 폴더가 있는지 확인
ls -la .claude/commands/

# 파일이 있는지 확인
ls .claude/commands/*.md
```

---

### 2. $ARGUMENTS가 치환 안 돼요!

**증상**: `/greet 철수`를 쳤는데 클로드가 "$ARGUMENTS"라는 글자를 그대로 출력

**원인과 해결**:

| 원인 | 잘못된 예 | 올바른 예 |
|------|----------|----------|
| 소문자로 적음 | `$arguments` | `$ARGUMENTS` |
| 대소문자 혼용 | `$Arguments` | `$ARGUMENTS` |
| 약어 사용 | `$args`, `$ARGS` | `$ARGUMENTS` |
| 중괄호 사용 | `${ARGUMENTS}` | `$ARGUMENTS` |
| 앞에 역슬래시 | `\$ARGUMENTS` | `$ARGUMENTS` |

**정답은 하나**:
```
$ARGUMENTS
```
→ 전부 대문자, 달러 기호 하나, 괄호 없음!

---

### 3. 스킬 실행 결과가 이상해요!

**증상**: 스킬이 실행되긴 하는데 원하는 결과가 아님

**원인과 해결**:

#### 지시가 모호한 경우

```markdown
❌ 모호한 지시:
코드를 봐주세요: $ARGUMENTS

✅ 구체적인 지시:
다음 파일의 코드를 리뷰해주세요: $ARGUMENTS

리뷰 기준:
1. 🔴 심각 — 버그, 보안 취약점
2. 🟡 주의 — 성능, 가독성 문제
3. 🟢 제안 — 개선 아이디어

출력 형식:
- 각 항목에 심각도 이모지 표시
- 해당 코드 줄 번호 표시
- 수정 제안 포함
```

#### 마크다운 문법 오류

```markdown
❌ 깨진 마크다운:
## 제목
- 리스트1
-리스트2 (하이픈 뒤 공백 없음)
```code (백틱 3개 필요)

✅ 올바른 마크다운:
## 제목
- 리스트1
- 리스트2
```​
코드 블록
```​
```

**팁**: 스킬 파일을 VS Code에서 미리보기로 확인해보세요!

### 4. 전역 스킬 vs 프로젝트 스킬 우선순위

**증상**: 같은 이름의 스킬이 두 곳에 있는데 어떤 게 실행되는지 모르겠음

**우선순위 규칙**:

```
1️⃣ 프로젝트 스킬 (최우선)
   → 내-프로젝트/.claude/commands/review.md

2️⃣ 전역 스킬
   → ~/.claude/commands/review.md
```

**확인 방법**:

```bash
# 프로젝트 스킬 확인
ls .claude/commands/

# 전역 스킬 확인
ls ~/.claude/commands/
```

**해결**: 같은 이름이 충돌하면:
- 프로젝트 전용이면 → 프로젝트 폴더에만 두기
- 모든 곳에서 쓰려면 → 전역 폴더에만 두기
- 프로젝트마다 다르게 → 프로젝트 폴더에 각각 두기

---

### 5. 스킬 파일 인코딩 문제

**증상**: 한글이 깨지거나 이상한 문자가 나옴

**원인과 해결**:

| 원인 | 해결 |
|------|------|
| 파일이 UTF-8이 아님 | UTF-8로 다시 저장 |
| BOM(Byte Order Mark) 포함 | BOM 없는 UTF-8로 저장 |
| 윈도우 메모장으로 저장 | VS Code 등 다른 편집기 사용 |

**VS Code에서 인코딩 확인**:
1. 하단 상태바에서 인코딩 표시 클릭
2. "Save with Encoding" 선택
3. "UTF-8" 선택

## 🛠️ 일반적인 디버깅 순서

문제가 생기면 이 순서로 확인하세요:

```
1️⃣ 폴더 위치 확인
   └── .claude/commands/ 에 파일이 있나?

2️⃣ 파일 확장자 확인
   └── .md로 끝나나?

3️⃣ $ARGUMENTS 확인
   └── 대문자로 정확히 적었나?

4️⃣ 마크다운 문법 확인
   └── 미리보기에서 정상적으로 보이나?

5️⃣ 인코딩 확인
   └── UTF-8로 저장했나?

6️⃣ 실행 위치 확인
   └── 프로젝트 루트에서 claude를 실행했나?
```

## ❓ 그래도 안 되면?

1. **클로드 코드 재시작**: 터미널을 닫고 다시 열어보세요
2. **`/help` 확인**: 스킬 목록에 나타나는지 체크
3. **간단한 스킬로 테스트**: 복잡한 스킬 대신 `greet.md`처럼 간단한 것으로 먼저 테스트
4. **파일 직접 확인**: `cat .claude/commands/스킬이름.md`로 내용 확인
