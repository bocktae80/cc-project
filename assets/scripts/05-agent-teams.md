# 05 — Agent Teams (~7분)

> Hook: AI 하나로 부족하면 팀으로!

---

## 1. Hook (30초)

[화면] 복잡한 프로젝트 구조

[나레이션]
웹사이트를 만든다고 생각해보세요.
디자인, 프론트엔드, 백엔드, 테스트...
한 사람이 다 하면 오래 걸리잖아요?

[화면] 여러 에이전트가 동시에 작업하는 애니메이션

[나레이션]
클로드 코드에서도 AI를 **팀으로** 만들 수 있어요!
각자 역할을 나눠서 동시에 일하는 거죠.

---

## 2. Concept (2분)

[전환] 학교 축제 비유

[화면] 비유 다이어그램
```
🏫 학교 축제 준비
│
├── 👤 반장 (Team Lead) — 전체 지휘
├── 🎨 미술부 (Designer) — 포스터 제작
├── 💻 컴퓨터부 (Developer) — 웹사이트 제작
└── ✅ 학급회장 (Tester) — 최종 검수
```

[나레이션]
학교 축제를 준비할 때처럼, 팀에는 **리더**와 **팀원**이 있어요.

리더는 전체 계획을 세우고, 할 일을 나눠주고, 진행 상황을 확인합니다.
팀원은 각자 맡은 일을 하고, 결과를 리더에게 보고해요.

[전환] 기술 다이어그램

[화면] 핵심 도구 3가지
```
TeamCreate → 팀 만들기
Task       → 할 일 배분 (에이전트 생성)
SendMessage → 대화하기
```

[나레이션]
**TeamCreate**로 팀을 만들고,
**Task**로 할 일을 만들어 에이전트에게 배분하고,
**SendMessage**로 서로 소통합니다.

[강조] 가장 중요한 것: 에이전트는 **병렬**로 동시에 작업!

---

## 3. Demo (3분 30초)

[화면] 터미널 — 빈 프로젝트

[행동] `claude` 실행

[나레이션]
TODO 웹앱을 만드는 시나리오를 실행해봅시다.

[행동] "팀을 만들어서 TODO 웹앱을 만들어줘. 프론트엔드, 백엔드, 테스터 3명으로" 입력

[화면] TeamCreate 실행 → 팀 생성

[나레이션]
팀이 만들어졌어요! 이제 클로드가 할 일 목록을 만듭니다.

[화면] TaskCreate로 3개 태스크 생성

[나레이션]
프론트엔드 개발, 백엔드 API, 통합 테스트... 3개의 할 일이 생겼어요.

[화면] 3개 에이전트가 동시에 작업 시작 (split view)

[강조] 3개가 **동시에** 진행되는 모습

[나레이션]
보이시나요? 세 에이전트가 동시에 각자의 일을 하고 있어요!
프론트엔드는 HTML을, 백엔드는 API를, 테스터는 테스트를 작성 중입니다.

[화면] SendMessage로 대화하는 모습

[나레이션]
에이전트끼리 메시지도 주고받아요.
"API 엔드포인트 뭐야?"라고 물으면 백엔드가 알려줍니다.

[화면] 최종 결과물 — 완성된 TODO 웹앱

[나레이션]
혼자 하면 30분 걸릴 일을, 팀으로 10분 만에 끝냈어요!
