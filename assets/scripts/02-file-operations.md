# 02 — 파일 읽기/쓰기 (~3분)

> Hook: 클로드가 파일을 직접 만든다고?

---

## 1. Hook (30초)

[화면] 코딩 중인 화면

[나레이션]
코딩하다 보면 파일을 열어보고, 새로 만들고, 수정하는 일이 정말 많죠?
클로드 코드에는 이걸 해주는 도구가 3개나 있어요.

[화면] Read, Write, Edit 아이콘 3개 등장

[나레이션]
비유하면 도서관 사서예요.
책을 읽어주고(Read), 새 책을 만들고(Write), 오탈자를 고치는(Edit) 사서!

---

## 2. Concept (1분)

[전환] 비교표 다이어그램

[화면] 3도구 비교표
```
📖 Read  = 읽기만! 안전해요
📝 Write = 새 파일 생성! (⚠️ 기존 파일 덮어쓰기 주의)
✏️ Edit  = 일부만 수정! 가장 안전한 수정 방법
```

[나레이션]
가장 중요한 차이!
**Write**는 파일을 **통째로 교체**해요. 기존 내용이 사라질 수 있어요.
**Edit**은 원하는 **부분만** 살짝 고쳐요.

[강조] Write = 통째로 교체 ⚠️ / Edit = 부분만 수정 ✅

[나레이션]
Edit에는 한 가지 규칙이 있어요.
old_string, 즉 "고칠 부분"이 파일에서 **딱 하나만** 있어야 해요.
같은 내용이 여러 개 있으면 어디를 고쳐야 할지 모르니까요!

---

## 3. Demo (1분)

[화면] 터미널 — 샘플 파일들이 있는 폴더

[행동] `claude` 실행 → "sample.txt 파일 읽어줘" 입력

[화면] Read 도구 실행, 파일 내용 표시

[나레이션]
Read 도구가 파일을 깔끔하게 보여줍니다.

[행동] "나의 첫 번째 프로그램.js 파일을 만들어줘" 입력

[화면] Write 도구 실행, 새 파일 생성

[행동] "before.js에서 claculateAverage를 calculateAverage로 고쳐줘" 입력

[화면] Edit 도구 실행, 함수명만 수정

[강조] old_string → new_string 부분만 변경된 모습

[나레이션]
Edit은 이렇게 필요한 부분만 정확하게 고칩니다!

---

## 4. Recap (30초)

[전환] 요약 화면

[나레이션]
오늘 배운 3가지!

1️⃣ **Read** = 파일 읽기 (안전, 가장 자주 사용)
2️⃣ **Write** = 새 파일 만들기 (덮어쓰기 주의!)
3️⃣ **Edit** = 부분만 수정 (old_string이 유일해야 함)

[화면] 다음 편 예고 — "03편: 10,000줄에서 원하는 코드 찾기"
