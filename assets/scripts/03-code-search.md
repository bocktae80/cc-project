# 03 — 코드 검색 (~3분)

> Hook: 10,000줄에서 원하는 코드, 어떻게 찾지?

---

## 1. Hook (30초)

[화면] 거대한 파일 목록이 스크롤되는 화면

[나레이션]
프로젝트가 커지면 파일이 수백 개, 코드가 수만 줄이 됩니다.
여기서 "TODO"가 적힌 줄을 찾으려면?
하나하나 열어볼 수는 없잖아요!

[화면] Glob과 Grep 아이콘 등장

[나레이션]
클로드에겐 두 가지 검색 도구가 있어요.
도서관에 비유하면, **카탈로그**와 **전문 검색**입니다!

---

## 2. Concept (1분)

[전환] 도서관 비유 다이어그램

[화면] 비유 화면
```
📚 도서관 카탈로그 (Glob) = "이런 제목의 책 어디 있어?"
🔍 전문 검색 엔진 (Grep) = "이 단어가 나오는 책은?"
```

[나레이션]
**Glob**은 **파일 이름**으로 찾아요.
"js 파일 다 보여줘", "test가 들어간 파일 찾아줘" 이런 식이죠.

**Grep**은 **파일 내용**에서 찾아요.
"TODO가 적힌 줄 찾아줘", "함수 정의 찾아줘" 이렇게요.

[강조] Glob = 파일명 검색 / Grep = 내용 검색

[나레이션]
실전에서는 둘을 **합쳐서** 쓰면 최강이에요!
먼저 Glob으로 범위를 좁히고, Grep으로 내용을 찾는 **깔때기 전략**!

---

## 3. Demo (1분)

[화면] playground 프로젝트 폴더

[행동] `claude` 실행 → "이 프로젝트에서 js 파일 모두 찾아줘" 입력

[화면] Glob 실행 → `**/*.js` 패턴으로 파일 목록

[나레이션]
Glob이 `**/*.js` 패턴으로 모든 자바스크립트 파일을 찾았어요!

[행동] "TODO가 적힌 줄을 모두 찾아줘" 입력

[화면] Grep 실행 → TODO 주석이 있는 파일과 줄번호

[강조] 파일명:줄번호 형식의 결과

[나레이션]
Grep이 5개 파일에서 TODO를 찾아냈어요.
줄번호까지 알려주니까 바로 찾아갈 수 있습니다!

[전환] 요약 화면
