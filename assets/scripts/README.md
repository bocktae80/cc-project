# 영상 대본 가이드

> 각 프로젝트의 영상 대본을 4단계 구조로 작성합니다.

## 대본 구조 (4단계)

### 1. Hook (30초)
- **목적**: 왜 이걸 알아야 하는가?
- **형식**: 질문 또는 문제 상황 제시
- **톤**: 호기심 자극, 공감

### 2. Concept (1~2분)
- **목적**: 핵심 개념 + 비유
- **형식**: 실생활 비유 → 기술 개념 매핑
- **시각**: 간단한 다이어그램/애니메이션

### 3. Demo (2~3분)
- **목적**: 화면 보며 따라하기
- **형식**: `[화면]`, `[행동]` 태그로 단계별 연출
- **팁**: 한 번에 하나씩, 천천히

### 4. Recap (30초)
- **목적**: 3줄 요약 + 다음 편 예고
- **형식**: 핵심 3가지 → 다음 영상 예고
- **톤**: 정리, 격려

---

## 태그 규칙

| 태그 | 설명 | 예시 |
|------|------|------|
| `[화면]` | 보여줄 화면 설명 | `[화면] 터미널에서 claude 실행` |
| `[행동]` | 사용자가 할 행동 | `[행동] "파일 읽어줘"라고 입력` |
| `[나레이션]` | 음성 해설 | `[나레이션] Read 도구가 실행되었습니다` |
| `[강조]` | 화면에서 강조할 부분 | `[강조] output_mode 파라미터` |
| `[전환]` | 화면 전환 | `[전환] 다이어그램으로 전환` |
